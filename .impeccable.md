# Impeccable Design Context

Condensed design context for AI-assisted development. For the full source of truth, see [`ARCANUM_STYLE_GUIDE.md`](ARCANUM_STYLE_GUIDE.md).

## Users

General users — indie developers, writers, hobbyist worldbuilders, and small teams building MUD game worlds and worldbuilding projects. They range from technically comfortable to complete beginners. They use Arcanum for creative sessions (hours at a time), switching between zone mapping, entity editing, config tuning, lore writing, and AI art generation. The tool has power-user features (command palette, keyboard shortcuts, dense editors) but must be approachable to someone opening it for the first time. Discoverability and guided onboarding matter as much as depth.

## Brand personality

**Cosmological. Precise. Ornate.**

The Arcanum is the Creator's own instrument — a window into the architecture of a world. The user operates it from a position above the world, with the same perspective the Creator once held. The interface should feel like it was carved into existence by the same force that shaped the land. Never a SaaS dashboard with a dark theme.

**Emotional goals:** grandeur, creative confidence, unhurried focus. The cosmos does not rush.

## Aesthetic direction

- **Theme:** Dark only. Midnight-teal deeps (`#001524` → `#15616d`), hearth-ember warm accent (`#ff7d00`), parchment text (`#ffecd1`). No light mode.
- **Typography:** Cinzel (display / headings), Crimson Pro (body), JetBrains Mono (code). All serif — no sans-serif anywhere in the UI. Dense type scale with 10px as the workhorse size for UI chrome.
- **Shape:** Large border-radius (14–18px) on panels, rounded-full pills for buttons and badges. Baroque softness, no sharp corporate rectangles.
- **Texture:** Decorative background images at low opacity (10–18%) with `mix-blend-soft-light` or gradient overlays on major surfaces. Panels use subtle gradient backgrounds, not flat fills.
- **Motion:** Slow and deliberate. 3-second ember pulse, 60-second rotations. `ease-unfurl` for entries, `ease-cosmic` for standard transitions. Respect `prefers-reduced-motion`.
- **Light:** Ember glow (`box-shadow`) on active and important elements, tide-teal ambient fill everywhere else. No neon. No hard drop shadows.

**Anti-references:** Generic AI-generated UIs (glassmorphism, gradient text, hero metrics, identical card grids, Inter / Geist fonts, teal-purple-pink palettes). Modern SaaS dashboards. Anything that reads as flat, corporate, or template-driven.

**References:** The baroque tradition — abundance, movement, the refusal to let surfaces rest passively. Celestial cartography. Illuminated manuscripts. Deep-ocean bioluminescence.

## Design principles

1. **Ember against the deep** — Warm accent light appears only where something is alive, active, or important. A single ember highlight against midnight-teal carries enormous weight. Don't dilute it.
2. **Density serves the builder** — This is a data-dense creative tool, not a marketing page. Small type (10px), tight spacing, and information-rich panels are appropriate. Progressive disclosure for secondary features, but never hide what power users need frequently.
3. **Texture over flatness** — Every major surface has visual texture: background images, gradient overlays, subtle glow. Flat solid-color panels feel dead. But texture must be quiet (low opacity, `pointer-events-none`) — never competing with content.
4. **Semantic tokens, not arbitrary values** — Colors, font sizes, tracking, gradients, and shadows come from the token system in `creator/src/index.css`. Hard-coded `rgba()` values and px sizes indicate a missing token, not a design choice.
5. **Accessibility is structural** — Keyboard navigation, focus traps on modals, ARIA labels, `prefers-reduced-motion` support, and sufficient contrast are baseline requirements, not polish items. The baroque aesthetic must not come at the cost of usability.

## Type scale

| Token | Size | Role |
|---|---|---|
| `text-3xs` | 10px | Sub-caption (room nodes, sprite cells) |
| `text-2xs` | 12px | UI chrome, badges, button labels, metadata |
| `text-xs` | 13px | Form inputs, secondary content |
| `text-sm` | 14px | Body text, descriptions |
| `text-base` | 15px | Default body (set on `<body>`) |
| `text-lg`+ | 18px+ | Section headings (with `font-display`) |

## Tracking scale

| Token | Value | Role |
|---|---|---|
| `tracking-label` | 0.18em | Badges, small uppercase labels |
| `tracking-ui` | 0.22em | Section headers, filter labels, metadata |
| `tracking-wide-ui` | 0.32em | Workspace titles, hero labels |
