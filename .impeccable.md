# Impeccable Design Context

## Design Context

### Users
Solo indie developers and small teams building MUD (text-based RPG) game worlds. They're technically capable — comfortable with YAML, game mechanics, zone graphs, stat formulas — but not professional designers. They use Arcanum for extended creative sessions (hours at a time), switching between zone mapping, entity editing, config tuning, and AI art generation. Speed and keyboard fluency matter; this is a power-user tool, not an onboarding-heavy consumer product.

### Brand Personality
**Cosmological. Precise. Ornate.**

The Arcanum is the Creator's own instrument — a window into the architecture of a world. The user operates it from a position above the world, with the same perspective the Creator once held. The interface should feel like it was carved into existence by the same force that shaped the land. Never a SaaS dashboard with a dark theme.

**Emotional goals**: Grandeur, creative confidence, unhurried focus. The cosmos does not rush.

### Aesthetic Direction
- **Theme**: Dark only. Deep indigo backgrounds (#22293c → #313a56), violet/lavender accents (#a897d2), no light mode.
- **Typography**: Cinzel (display/headings), Crimson Pro (body), JetBrains Mono (code). All serif — no sans-serif anywhere in the UI. Dense type scale with 10px as the workhorse size for UI chrome.
- **Shape**: Large border-radius (24–36px) on panels, rounded-full pills for buttons and badges. Baroque softness, no sharp corporate rectangles.
- **Texture**: Decorative background images at low opacity (6–18%) with `mix-blend-screen` or gradient overlays on major surfaces (sidebar, toolbar, tab bar, zone map). Panels use subtle gradient backgrounds, not flat fills.
- **Motion**: Slow and deliberate. 3–4 second pulse animations, 60-second rotations. `ease-unfurl` for entries, `ease-cosmic` for standard transitions. Respect `prefers-reduced-motion`.
- **Light**: Gold/violet glow (`box-shadow`) on active and important elements. No neon. No hard drop shadows.

**Anti-references**: Generic AI-generated UIs (glassmorphism, gradient text, hero metrics, identical card grids, Inter/Geist fonts, teal-purple-pink palettes). Modern SaaS dashboards. Anything that reads as flat, corporate, or template-driven.

**References**: The baroque tradition — abundance, movement, the refusal to let surfaces rest passively. Celestial cartography. Illuminated manuscripts.

### Design Principles

1. **Gold against the void** — Warm accent light appears only where something is alive, active, or important. A single gold highlight against deep indigo carries enormous weight. Don't dilute it.

2. **Density serves the builder** — This is a data-dense creative tool, not a marketing page. Small type (10px), tight spacing, and information-rich panels are appropriate. Progressive disclosure for secondary features, but never hide what power users need frequently.

3. **Texture over flatness** — Every major surface has visual texture: background images, gradient overlays, subtle glow. Flat solid-color panels feel dead. But texture must be quiet (low opacity, pointer-events-none) — never compete with content.

4. **Semantic tokens, not arbitrary values** — Colors, font sizes, tracking, gradients, and shadows come from the design token system in `index.css`. Hard-coded rgba values and px sizes indicate a missing token, not a design choice.

5. **Accessibility is structural** — Keyboard navigation, focus traps on modals, ARIA labels, `prefers-reduced-motion` support, and sufficient contrast are baseline requirements, not polish items. The baroque aesthetic must not come at the cost of usability.

### Type Scale
| Token | Size | Role |
|-------|------|------|
| `text-3xs` | 9px | Sub-caption (room nodes, sprite cells) |
| `text-2xs` | 10px | UI chrome, badges, button labels, metadata |
| `text-xs` | 12px | Form inputs, secondary content |
| `text-sm` | 14px | Body text, descriptions |
| `text-base` | 15px | Default body (set on `<body>`) |
| `text-lg`+ | 18px+ | Section headings (with `font-display`) |

### Tracking Scale
| Token | Value | Role |
|-------|-------|------|
| `tracking-label` | 0.18em | Badges, small uppercase labels |
| `tracking-ui` | 0.22em | Section headers, filter labels, metadata |
| `tracking-wide-ui` | 0.32em | Workspace titles, hero labels |

### Gradient Utilities
| Class | Role |
|-------|------|
| `bg-gradient-active` | Selected/active tab or item background |
| `bg-gradient-active-strong` | Emphasized active state, hover |
| `bg-gradient-panel` | Panel/card background |
| `bg-gradient-panel-light` | Lighter panel variant |
| `bg-gradient-glow-top` | Top-of-panel ambient violet glow |
