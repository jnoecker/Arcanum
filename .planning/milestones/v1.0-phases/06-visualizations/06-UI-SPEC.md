# Phase 6 -- UI Design Contract

> Visual and interaction contract for the Visualization layer of the Tuning Wizard. Three Recharts-based charts (XP curve, mob tier power, stat profile radar) added below metric cards. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|

| Preset | not applicable |

## Spacing Scale

Declared values (must be multiples of 4). Inherited from Phase 4 -- two phase-specific additions for chart card internals.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Legend icon gap, chart margin top/bottom |
| sm | 8px | Chart card title bottom margin, level selector gap |
| md | 16px | Chart card internal padding (p-4), gap between chart cards (gap-4) |
| lg | 24px | Horizontal padding on ChartRow container (px-6) |
| xl | 32px | (Inherited -- no new use in Phase 6) |
| 2xl | 48px | (Inherited -- no new use in Phase 6) |
| 3xl | 64px | (Inherited -- no new use in Phase 6) |

Exceptions: Recharts `margin` prop uses `{ top: 5, right: 20, bottom: 5, left: 10 }` -- these are SVG-internal offsets controlled by Recharts, not CSS spacing. The values are odd numbers because they are pixel offsets for axis label clearance, not layout spacing.

Source: Phase 4 UI-SPEC; CONTEXT.md D-03; 06-RESEARCH.md Pattern 3.

## Typography

4 sizes, 2 weights. Consistent with Phase 4 type scale. SVG text rendered via Recharts tick/style props (not Tailwind classes -- SVG elements do not support Tailwind utilities).

| Role | Font | Size | Weight | Line Height | Rendering |
|------|------|------|--------|-------------|-----------|
| Chart card title | Cinzel | 14px | 400 | 1.3 | Tailwind: `font-display text-[14px] font-normal uppercase tracking-[0.5px] text-text-secondary` |
| Axis tick labels | Crimson Pro | 12px | 400 | n/a | SVG prop: `tick={{ fill: "#95a0bf", fontSize: 12, fontFamily: "'Crimson Pro', Georgia, serif" }}` |
| Radar angle labels | Crimson Pro | 11px | 400 | n/a | SVG prop: `tick={{ fill: "#95a0bf", fontSize: 11, fontFamily: "'Crimson Pro', Georgia, serif" }}` |
| Legend text | Crimson Pro | 13px | 400 | 1.4 | Recharts Legend uses default browser rendering; font inherited from chart container |
| Level selector label | Crimson Pro | 13px | 400 | 1.5 | Tailwind: `font-sans text-[13px] text-text-muted` |
| Level selector dropdown | Crimson Pro | 13px | 400 | 1.5 | Tailwind: `ornate-input` class (existing pattern) |

**Size scale (4 values):** 11px, 12px, 13px, 14px
**Weight scale (2 values):** 400 (regular) only -- no bold/semibold in chart context

Rationale: Chart typography is intentionally smaller than the workspace body text. 11px and 12px are acceptable for SVG axis labels rendered in Crimson Pro at chart scale. 14px matches the MetricCard section title from Phase 4.

Source: CONTEXT.md D-07; 06-RESEARCH.md Pitfall 2; Phase 4 UI-SPEC typography scale.

All chart colors are passed as hex strings to Recharts SVG props (`stroke`, `fill`, `style`). Tailwind classes do not apply to SVG elements.

Defined as a `const` object in `src/lib/tuning/chartColors.ts`:

| Constant | Hex | Source Token | Purpose |
|----------|-----|-------------|---------|
| `currentSeries` | #aebada | `text-text-secondary` | Line stroke, bar fill, radar stroke+fill for current config |
| `presetSeries` | #c8a46a | `warm` | Line stroke, bar fill, radar stroke+fill for preset config |
| `grid` | #39455f | `border-muted` | CartesianGrid stroke, PolarGrid stroke |
| `axisText` | #95a0bf | `text-muted` | XAxis/YAxis/PolarAngleAxis tick fill |
| `axisLine` | #56617d | `border-default` | XAxis/YAxis stroke (axis line itself) |
| `barHp` | #c05060 | `chart-hp` (existing token in index.css) | Mob tier HP bars |
| `barDamage` | #dbb8b8 | `status-error` | Mob tier damage bars |

### Accent Reserved For

- **#c8a46a (warm/preset series):** Preset data line in XP curve, preset radar polygon fill+stroke. This is the primary visual signal -- what the builder is evaluating.
- **#aebada (current series):** Current config data line in XP curve, current radar polygon fill+stroke. Muted to recede behind preset.
- **#c05060 (chart-hp):** Mob HP bars only. Already exists as `--color-chart-hp` in index.css.
- **#dbb8b8 (status-error):** Mob damage bars only. Reused from existing status token.
- **#8caec9 (stellar-blue):** Mob armor bars only. Reused from existing cool accent token.
- **#a3c48e (status-success):** Mob XP bars only. Reused from existing status token.

## Focal Point

**Primary visual anchor for Phase 6:** The three chart cards as a horizontal row. When a preset is selected, the chart row sits between the metric summary cards and the health check banner/search bar. The XP curve line chart is the most immediately legible (two crossing lines), making it the natural first-scan target. The warm gold preset line dominates visually against the muted current line.

**Secondary anchor:** The mob tier bar chart level selector dropdown. This is the only interactive element within the chart row -- it draws the eye because it affords interaction in an otherwise static display area.

## Component Inventory

### New: ChartRow

- **Purpose:** Container for the 3 chart cards. Conditional render -- only when a preset is selected.
- **Layout:** `grid grid-cols-3 gap-4 px-6 mt-4 mb-4`
- **Visibility:** Rendered conditionally: `{selectedPresetId && currentMetrics && activePresetMetrics && presetConfig && <ChartRow ... />}`
- **Entrance animation:** `animate-unfurl-in` (existing Tailwind utility -- same as MetricSectionCards in Phase 4)
- **Placement in TuningWizard:** Between `MetricSectionCards` and `HealthCheckBanner`.
- **Props:**

  ```typescript
    presetConfig: AppConfig;
    currentMetrics: MetricSnapshot;
    presetMetrics: MetricSnapshot;
  }
  export function ChartRow({ currentConfig, presetConfig, currentMetrics, presetMetrics }: ChartRowProps)
  ```
- **File:** `creator/src/components/tuning/charts/ChartRow.tsx`

### New: XpCurveChart

- **Purpose:** Recharts LineChart comparing XP-per-level for current vs preset configs across levels 1-50.
- **Card surface:** `bg-bg-tertiary rounded-lg border border-border-muted p-4`
- **Title:** "XP CURVE" in Cinzel 14px weight 400 uppercase, `text-text-secondary`, `tracking-[0.5px]`, 8px bottom margin.
- **Chart:** `<ResponsiveContainer width="100%" height={220}>` wrapping `<LineChart>`.
- **Data:** Array of 50 objects `{ level: number; current: number; preset: number }` produced by `buildXpCurveData()`.
- **Lines:**
  - Current: `stroke={CHART_COLORS.currentSeries}`, `strokeWidth={2}`, `dot={false}`, `type="monotone"`
  - Preset: `stroke={CHART_COLORS.presetSeries}`, `strokeWidth={2}`, `dot={false}`, `type="monotone"`
- **Axes:**
  - XAxis: `dataKey="level"`, tick style per typography table above, stroke `axisLine`.
  - YAxis: tick style per typography table, `tickFormatter` abbreviating values >= 1000 as `{n}k`.
- **Grid:** `<CartesianGrid strokeDasharray="3 3" stroke={CHART_COLORS.grid} />`
- **Tooltip:** `<Tooltip />` (Recharts default -- D-11).
- **Legend:** `<Legend />` (Recharts default positioning -- bottom of chart).
- **Animation:** `isAnimationActive={true}`, `animationDuration={300}`, `animationEasing="ease-out"` on both Lines.
- **Props:**
  ```typescript
  interface XpCurveChartProps {
    data: Array<{ level: number; current: number; preset: number }>;
  }
  ```
- **File:** `creator/src/components/tuning/charts/XpCurveChart.tsx`

### New: MobTierChart

- **Purpose:** Recharts BarChart showing HP, damage, armor, XP across all 4 mob tiers at a user-selected level.
- **Card surface:** `bg-bg-tertiary rounded-lg border border-border-muted p-4`
- **Title row:** "MOB POWER" in Cinzel 14px weight 400 uppercase, `text-text-secondary`, followed by a level selector dropdown aligned right.
- **Level selector:**
  - `<select>` element with options: Lv 10, Lv 20, Lv 30, Lv 50.
  - Default: Lv 30 (D-09).
  - Styling: `ornate-input` class, `text-[13px]`, `py-0.5 px-2`.
  - State: local `useState<number>(30)` -- no store needed.
- **Chart:** `<ResponsiveContainer width="100%" height={220}>` wrapping `<BarChart>`.
- **Data:** Array of 4 objects `{ tier: string; hp: number; damage: number; armor: number; xp: number }` produced by `buildMobTierData()`.
  - Tier display labels: "Weak", "Standard", "Elite", "Boss" (mapping from config keys `weak`, `standard`, `elite`, `boss`).
- **Bars:** 4 `<Bar>` elements, grouped (Recharts default grouping when multiple Bars share an XAxis):
  - `dataKey="hp"`, `name="HP"`, `fill={CHART_COLORS.barHp}`
  - `dataKey="damage"`, `name="Damage"`, `fill={CHART_COLORS.barDamage}`
  - `dataKey="armor"`, `name="Armor"`, `fill={CHART_COLORS.barArmor}`
  - `dataKey="xp"`, `name="XP"`, `fill={CHART_COLORS.barXp}`
- **Bar category gap:** `barCategoryGap="20%"` for visual breathing room between tier groups.
- **Axes:** XAxis `dataKey="tier"`, YAxis with `tickFormatter` for large number abbreviation.
- **Grid:** `<CartesianGrid strokeDasharray="3 3" stroke={CHART_COLORS.grid} />`
- **Tooltip:** `<Tooltip />` (Recharts default).
- **Legend:** `<Legend />` (Recharts default positioning).
- **Animation:** `isAnimationActive={true}`, `animationDuration={300}`, `animationEasing="ease-out"` on all Bars.
- **Props:**
  ```typescript
  interface MobTierChartProps {
    currentConfig: AppConfig;
  }
  ```
  Note: This chart shows absolute values at a level (not current vs preset comparison). It visualizes the preset config's mob tier data since the builder is evaluating the preset. The `currentConfig` prop is the merged preset config.
- **File:** `creator/src/components/tuning/charts/MobTierChart.tsx`

### New: StatRadarChart

- **Purpose:** Recharts RadarChart comparing current vs preset stat scaling effectiveness across all stat bindings.
- **Card surface:** `bg-bg-tertiary rounded-lg border border-border-muted p-4`
- **Title:** "STAT PROFILE" in Cinzel 14px weight 400 uppercase, `text-text-secondary`, 8px bottom margin.
- **Chart:** `<ResponsiveContainer width="100%" height={220}>` wrapping `<RadarChart cx="50%" cy="50%" outerRadius="70%">`.

  - `<PolarRadiusAxis tick={false} axisLine={false} />` -- hidden radius axis (values are heterogeneous units, numeric labels would be misleading).
- **Tooltip:** `<Tooltip />` (Recharts default).
- **Legend:** `<Legend />` (Recharts default positioning).
- **Animation:** `isAnimationActive={true}`, `animationDuration={300}` on both Radar elements.
- **Props:**

## Interaction Contracts

### Chart Row Appearance

1. No preset selected: ChartRow is absent from DOM. Wizard shows only preset cards and parameter browser.
2. User selects a preset: ChartRow renders with `animate-unfurl-in` (300ms ease). All 3 charts populate with data computed via `useMemo` from current and preset configs.
3. User selects a different preset: Chart data updates. Recharts animates line morphing, bar resizing, and radar polygon reshaping over 300ms ease-out (D-06). No DOM remount -- same component instances with new data props.
4. User deselects preset (clicks selected card again): ChartRow unmounts from DOM.

### Mob Tier Level Selector

1. Default state: Dropdown shows "Lv 30" selected (D-09).
2. User opens dropdown: Shows 4 options -- Lv 10, Lv 20, Lv 30, Lv 50.
3. User selects a different level: Bar chart data recomputes via `useMemo`. Bars animate to new heights over 300ms.
4. Dropdown state is local to MobTierChart component (`useState`). No persistence needed -- resets to Lv 30 when ChartRow remounts.

### Chart Tooltip (Recharts Default)

1. User hovers over a data point (line dot, bar, radar vertex): Recharts built-in tooltip appears showing exact values.
2. Format: Default Recharts tooltip -- series name + value. No custom styling for v1 (D-11).
3. Mouse leaves data point: Tooltip disappears immediately.

### Responsive Behavior

1. Charts use `<ResponsiveContainer width="100%" height={220}>` -- they fill their grid cell width.
2. If the wizard panel is narrow (e.g., user has a narrow window), `grid-cols-3` will compress each chart. At extreme narrow widths, axis labels may overlap. This is acceptable for v1 -- the wizard is designed for a 1024px minimum window width (per Tauri config).
3. No breakpoint switching (no `grid-cols-1` fallback) -- the Tauri window enforces minimum dimensions.

## Data Preparation Contract

### buildXpCurveData

- **Input:** `currentConfig: AppConfig`, `presetConfig: AppConfig`
- **Output:** 50-element array of `{ level, current, preset }` for levels 1-50.
- **Computation:** Calls `xpForLevel(level, config.progression.xp)` for each level on both configs.
- **Edge cases:** None expected -- xpForLevel is a pure arithmetic function.

### buildMobTierData

- **Input:** `config: AppConfig`, `level: number`
- **Output:** 4-element array of `{ tier, hp, damage, armor, xp }` for tiers weak/standard/elite/boss.
- **Computation:**
  - `hp`: `mobHpAtLevel(config.mobTiers[tier], level)`
  - `damage`: `mobAvgDamageAtLevel(config.mobTiers[tier], level)`
  - `armor`: `config.mobTiers[tier].baseArmor` (armor is not level-scaled in MobTierConfig)
  - `xp`: `config.mobTiers[tier].baseXpReward + config.mobTiers[tier].xpRewardPerLevel * level`
- **Tier label mapping:** `{ weak: "Weak", standard: "Standard", elite: "Elite", boss: "Boss" }`

### buildStatRadarData

- **Input:** `currentBindings: StatBindings`, `presetBindings: StatBindings`
- **Output:** 8-element array of `{ stat, current, preset }`.
- **Normalization:** Divisor fields are inverted (`1 / divisor`) so "bigger polygon = more powerful".
  - Inverted: `meleeDamageDivisor`, `spellDamageDivisor`, `hpScalingDivisor`, `manaScalingDivisor`
  - Direct: `dodgePerPoint`, `hpRegenMsPerPoint`, `manaRegenMsPerPoint`, `xpBonusPerPoint`
- **Note:** Raw values are used (no min-max normalization). The radar auto-scales each axis independently. If the shape is unreadable due to wildly different magnitudes, normalization can be added as a follow-up (per 06-RESEARCH.md Open Question 2).
