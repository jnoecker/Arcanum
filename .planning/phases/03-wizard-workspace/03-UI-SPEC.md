### Accent Reserved For

- **Warm gold** (`text-warm` / `border-warm`): Selected preset card glow/border, workspace title, active filter chip border
- **Violet accent** (`text-accent`): Search input focus ring, parameter row hover highlight, section field count badge
- **Status colors**: `text-status-success` (#a3c48e) for preset value decreases that are beneficial, `text-status-warning` (#bea873) for increases, `text-status-info` (#8caec9) for neutral changes

### Preset Card Accent Colors (D-03)

| Preset | Border/Glow Color | Rationale |
|--------|-------------------|-----------|
| Casual Adventure | `--color-warm` (#c8a46a) warm gold | Warm, inviting, relaxed feel |
| Balanced Challenge | `--color-stellar-blue` (#8caec9) cool silver-blue | Neutral, measured, equilibrium |
| Hardcore Grind | `--color-status-error` (#dbb8b8) muted crimson-rose | Danger, intensity, high stakes |

Unselected cards: `border-border-muted` (#39455f) at 100% opacity.
Selected card: preset accent color border at 100% + `box-shadow` glow at 35% opacity, 20px spread.
Dimmed cards (when one is selected): opacity 0.65 on unselected cards.

## Component Inventory

### 1. TuningWizard (workspace root)

- **Layout**: Full-height flex column with `min-h-0` on parent. Scrollable main area.
- **Structure**: Title bar (workspace title) -> Preset selector row -> Search/filter bar (sticky) -> Parameter browser (scrollable)
- **Top padding**: 64px (3xl) from workspace top to first element, 24px (lg) horizontal padding
- **Title**: "Tuning Wizard" in Cinzel 22px weight 400, color `text-text-primary`

### 2. PresetCard

- **Layout**: Three equal-width cards in a horizontal flex row, centered. Gap: 32px (xl). Max card width: 320px.
- **Card surface**: `bg-bg-tertiary` with `rounded-xl` (12px radius), 24px (lg) internal padding
- **Card border**: 1px solid preset accent color (see table above) when selected; `border-border-muted` when unselected
- **Content stack** (top to bottom, 16px gap):
  1. Preset name: Cinzel 18px weight 600, color `text-text-primary`
  2. Description: Crimson Pro 15px weight 400, color `text-text-secondary`, 2 lines max with line-clamp
  3. Key metric indicators: 3-4 rows of label-value pairs. Label in Crimson Pro 13px `text-text-muted`, value as a pill/badge with preset accent color background at 14% opacity and accent text color
- **Interaction**:
  - Hover: `bg-bg-hover` background transition 200ms
  - Click: selects preset, applies selected state
  - Selected: accent border + glow, parameter browser populates
  - Unselected (when another is selected): opacity 0.65, transition 300ms
  - Cursor: `cursor-pointer`

### 3. SearchFilterBar

- **Position**: Sticky, `top-0`, `z-10`, with `bg-bg-primary` background + bottom border `border-border-muted`
- **Layout**: Flex row. Search input on left (flex-1), section filter chips on right. Gap: 16px (md).
- **Margin**: 48px (2xl) top margin from preset cards, 24px (lg) bottom margin to parameter list
- **Search input**: `.ornate-input` class, Crimson Pro 15px, placeholder "Search parameters...", border `border-border-default`, focus ring `border-border-focus`
- **Debounce**: 150ms debounce on search input before filtering
- **Filter chips**: One per TuningSection (4 total). Crimson Pro 13px weight 500 uppercase with 0.18em letter-spacing. Padding: 4px vertical, 12px horizontal. Rounded-full.
  - Active: `bg-accent` at 14% opacity, `text-accent`, border 1px `border-accent-subtle` (rgba(168,151,210,0.35))
  - Inactive: `bg-bg-secondary`, `text-text-muted`, border 1px `border-border-muted`
  - Click: toggles section on/off
  - All active by default

### 4. ParameterSection (collapsible)

- **Header**: Flex row. Left: Cinzel 14px uppercase weight 400 `text-text-secondary` with 0.5px letter-spacing. Right: field count badge in Crimson Pro 13px `text-accent` with `bg-accent` at 14% opacity, rounded-full, 4px/8px padding.
- **Collapse**: Chevron icon (simple CSS triangle or inline SVG) rotates 90 degrees on expand/collapse. Transition 200ms.
- **Default state**: All sections expanded (D-11 discretion: all expanded aids first-time exploration)
- **Section gap**: 32px (xl) between sections
- **Separator**: 1px `border-border-muted` line above each section header

### 5. ParameterRow

- **Layout**: CSS grid with 3 columns (no preset selected) or 4 columns (preset selected). Grid template: `[label 1fr] [current 120px] [preset 120px] [description 1.5fr]` when preset active; drop preset column otherwise.
- **Row height**: Auto, min 40px. Vertical padding: 8px (sm).
- **Alternating rows**: Even rows `bg-bg-secondary` at 50% opacity for readability.
- **Label**: Crimson Pro 15px weight 500 `text-text-primary`
- **Current value**: JetBrains Mono 14px weight 400 `text-text-secondary`, right-aligned
- **Preset value** (when preset selected): JetBrains Mono 14px weight 400, right-aligned. Color varies by diff status:
  - Unchanged: `text-text-muted`
  - Changed (increase): `text-status-warning` (#bea873)
  - Changed (decrease): `text-status-info` (#8caec9)
- **Description**: Crimson Pro 13px weight 400 `text-text-muted`, single line with text-overflow ellipsis
- **Hover**: Full row background transitions to `bg-bg-hover` at 200ms
- **Changed row highlight**: When preset is selected and value differs, left border 2px solid preset accent color

### 6. EmptyState (search yields no results)

- **Centered** in parameter browser area
- **Heading**: Crimson Pro 18px weight 500 `text-text-secondary`
- **Body**: Crimson Pro 15px weight 400 `text-text-muted`

## Interaction Contracts

### Preset Selection Flow

1. Initial state: Three cards visible, none selected. Parameter browser shows all 137 fields with current values only.
2. User clicks a preset card: Card gains selected glow/border. Other cards dim to 0.65 opacity. Parameter browser gains "Preset Value" column. Changed rows gain color-coded preset values and left border highlight. Browser smooth-scrolls into view if not visible.
3. User clicks a different preset: Selection transfers. Transition: previous card dims, new card glows. Parameter values update.
4. User clicks the selected preset again: Deselects. All cards return to default state. Preset column removed from parameter browser.

### Search/Filter Flow

1. User types in search box: After 150ms debounce, parameter rows filter to show only fields where `label`, `description`, or `path` matches the query (case-insensitive substring match). Sections with zero matching fields collapse and hide.
2. User clicks a section filter chip: Toggles that section. Sections toggled off hide completely. Multiple can be active. Interaction is instant (no debounce).
3. Search + filter combine: Both conditions apply (AND logic). A field must match the search query AND belong to an active section to appear.
4. Clearing search: Show all fields in active sections.

### Scroll Behavior

- Workspace root is the scroll container (not document body).
- Search/filter bar is sticky at top of scroll container.
- Preset cards scroll away above the sticky bar.

## State Contract

New `tuningWizardStore` Zustand store (D-07):

| State Field | Type | Default | Purpose |
|-------------|------|---------|---------|
| selectedPresetId | `string \| null` | `null` | Currently selected preset ID |
| searchQuery | `string` | `""` | Current search input value |
| activeSections | `Set<TuningSection>` | all 4 sections | Active section filters |
| collapsedSections | `Set<TuningSection>` | empty | Manually collapsed sections |

Store is session-only (no localStorage persistence). State survives tab switches within the session (D-07).
