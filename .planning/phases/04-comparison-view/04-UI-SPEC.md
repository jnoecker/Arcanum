## Typography

4 sizes, 2 weights. Inherited from Phase 3 with one addition for metric card section title.

| Role | Font | Size | Weight | Line Height | Letter Spacing |
|------|------|------|--------|-------------|----------------|
| Card section title | Cinzel | 14px | 400 | 1.3 | 0.5px, uppercase |
| Metric row label | Crimson Pro | 14px | 400 | 1.5 | normal |
| Metric value (current) | JetBrains Mono | 13px | 400 | 1.4 | normal |
| Metric value (preset) + delta | JetBrains Mono | 13px | 400 | 1.4 | normal |
| Delta percentage badge | JetBrains Mono | 13px | 400 | 1.0 | normal |

| Changes-count badge | Crimson Pro | 13px | 400 | 1.0 | normal |

**Size scale (4 values):** 13px, 14px, 15px, 22px (22px is the workspace title, inherited)
**Weight scale (2 values):** 400 (regular), 600 (emphasis — parameter labels only)

Rationale: 13px is added for compact numeric values inside metric cards and tooltips; it sits below the Phase 3 minimum of 14px but is appropriate for JetBrains Mono numerics at high density. 14px covers section labels and metric row labels. 15px is the standard parameter label. No new display sizes are introduced.

### Color Roles for Diff Indicators (D-08)

Phase 4 corrects Phase 3's diff color semantics. Phase 3 used `text-status-warning` (orange) for increase and `text-status-info` (blue) for decrease. Phase 4 mandates:

| Diff Direction | Token | Value | Applies To |
|----------------|-------|-------|------------|
| Increase (▲) | `text-status-success` | #a3c48e | ParameterRow preset column, MetricCard delta badge |
| Decrease (▼) | `text-status-error` | #dbb8b8 | ParameterRow preset column, MetricCard delta badge |
| Unchanged (—) | `text-text-muted` | #95a0bf | ParameterRow preset column, MetricCard delta badge |

Rationale: Green for increase, muted red for decrease. No value judgment is implied — this is directional only (D-08). The color pairing is direction-only, not benefit/harm.

### Accent Reserved For

- **`text-status-success` (#a3c48e):** Increase indicators (▲ deltas) in MetricCards and ParameterRow; "changed" badge on collapsed ParameterSection headers.
- **`text-status-error` (#dbb8b8):** Decrease indicators (▼ deltas) in MetricCards and ParameterRow.
- **`text-text-muted` (#95a0bf):** Unchanged values (— indicator) in MetricCards and ParameterRow.
- **`text-accent` (#a897d2):** Section field-count badge (inherited from Phase 3 ParameterSection header).
- **`text-warm` (#c8a46a):** Workspace title, selected preset card glow (inherited from Phase 3).
- **Impact badge HIGH:** `text-status-error` (#dbb8b8) — high-impact field tooltip badge.
- **Impact badge MEDIUM:** `text-status-warning` (#bea873) — medium-impact field tooltip badge.
- **Impact badge LOW:** `text-text-muted` (#95a0bf) — low-impact field tooltip badge.

## Focal Point

**Primary visual anchor for Phase 4:** The MetricSectionCards block. When a preset is selected, the 2x2 grid of KPI cards becomes the visual center of gravity — it sits between the preset cards and the parameter browser, and carries the derived-metric comparison that is the core value of the wizard. Delta badges (▲/▼ with percentage) in success/error colors are the primary scan target within each card.

**Secondary anchor:** The changes-count badge on collapsed ParameterSection headers. When the builder wants to drill in, the badge tells them which sections have changed fields without requiring expansion.

## Component Inventory

### New: MetricSectionCards

- **Purpose:** 2x2 grid of KPI cards, one per TuningSection. Appears between the preset row and the SearchFilterBar only when a preset is selected (D-06).
- **Layout:** `grid grid-cols-2 gap-4 px-6 mt-6 mb-6` — 2 columns, 4px gap between cards, 24px horizontal padding
- **Visibility:** Rendered conditionally — `{selectedPresetId && <MetricSectionCards ... />}`. Entrance animation: `animate-unfurl-in` (existing Tailwind utility in index.css) — fade + scale from 0.97, 300ms, `ease-unfurl`.
- **Props:** `currentMetrics: MetricSnapshot`, `presetMetrics: MetricSnapshot`
- **Component signature:**
  ```typescript
  interface MetricSectionCardsProps {
    currentMetrics: MetricSnapshot;
    presetMetrics: MetricSnapshot;
  }
  export function MetricSectionCards({ currentMetrics, presetMetrics }: MetricSectionCardsProps)
  ```

### New: MetricCard

- **Purpose:** Single KPI card for one TuningSection. Shows section name + 2-3 curated metric rows with before/after values and delta badges.
- **Surface:** `bg-bg-secondary rounded-xl p-4 border border-border-muted`
- **Header:** Section name in Cinzel 14px weight 400 uppercase, `text-text-secondary`, 0.5px letter-spacing, 8px bottom margin.
- **Metric rows:** See MetricRow definition below.
- **Curated metrics per card (D-04):**

| Card | Metric 1 | Metric 2 | Metric 3 |
|------|----------|----------|----------|
| Combat & Stats | Mob HP (Standard, Lv10) | Mob HP (Standard, Lv30) | Dodge Chance (Lv10) |
| Progression & Quests | XP to Level 10 | XP to Level 30 | Player HP (Lv10) |
| Economy & Crafting | Gold/Kill (Standard, Lv10) | Gold/Kill (Standard, Lv30) | — (2 metrics + note) |
| World & Social | Regen Interval (base) | — (1 metric + changed-count note) | — |

Economy note: Buy/sell multipliers are raw config values (not in MetricSnapshot). Show "Buy/sell multipliers visible in raw fields below" as muted caption below the 2 metrics.

World note: If only 1 meaningful derived metric exists, show regen interval + "{N} other fields changed" in `text-text-muted` caption instead of padding with uninformative values.

Data sources:
- `MetricSnapshot.mobHp["standard"][level]` → Mob HP
- `MetricSnapshot.dodgeChance[level]` → Dodge Chance
- `MetricSnapshot.xpPerLevel[level]` → XP to level
- `MetricSnapshot.playerHp[level]` → Player HP
- `MetricSnapshot.mobGoldAvg["standard"][level]` → Gold/Kill
- `MetricSnapshot.regenInterval[1]` → Regen Interval (level 1 base value)

### New: MetricRow (internal to MetricCard)

- **Layout:** `flex items-baseline justify-between gap-2 py-1`
- **Label:** Crimson Pro 14px weight 400, `text-text-secondary`, cursor-default. Tippy tooltip attached (D-12) — formula explanation.
- **Current value:** JetBrains Mono 13px weight 400, `text-text-muted`, right-aligned.
- **Preset value + delta:** JetBrains Mono 13px weight 400, right-aligned. Format: `{formattedValue} ▲ X.X%` or `{formattedValue} ▼ X.X%` or `{formattedValue} —`. Color follows diff direction tokens above.
- **Formula tooltip (D-12):** Tippy instance on label element, `theme: "arcanum"`, `placement: "top-start"`, `delay: [200, 100]`, `maxWidth: 260`. Content is a human-readable formula explanation string.

Formula tooltip content by metric:
| Metric | Formula copy |
|--------|-------------|
| Mob HP (Normal) | "baseHp + hpPerLevel × level (Normal tier)" |
| Dodge Chance | "agility stat bonus / (agility stat bonus + 100)" |
| XP to Level N | "Cumulative XP from level 1 to N using the XP exponent curve" |
| Player HP | "baseHp + hpPerLevel × level" |
| Gold/Kill (Normal) | "mob base gold × goldMultiplier (Normal tier)" |
| Regen Interval | "regenIntervalBase (milliseconds per regen tick)" |

### Modified: ParameterRow (tooltip + enhanced diff display)

**Change 1 — Tooltip on field label (UI-04, D-10, D-11):**
- Attach Tippy instance to the label `<span>` via `useRef` + `useEffect`.
- Config: `theme: "arcanum"`, `placement: "top-start"`, `delay: [250, 100]`, `maxWidth: 280`.
- Tooltip content built from FieldMeta:
  1. `meta.description` — always present, Crimson Pro 13px.
  2. `meta.interactionNote` — render as "Interacts with: {note}" when defined.
  3. Impact badge: "HIGH IMPACT" / "MEDIUM IMPACT" / "LOW IMPACT" in respective color.
- Tooltip content is built as an HTML string using `allowHTML: true` Tippy option.

**Change 2 — Arrow + percentage delta (D-09):**
- When `isChanged` is true and both `currentValue` and `presetValue` are numeric:
  - Render: `{presetValue formatted} {▲ or ▼} {pct}%` in a single `<span>` in the preset column.
  - Color: `text-status-success` for increase, `text-status-error` for decrease.
- When `isChanged` is false: render preset value in `text-text-muted` with no arrow.
- When either value is non-numeric but changed: render preset value in `text-status-success` with no percentage (direction unknown for non-numeric types).
- Edge case — zero denominator: use `pctDelta()` utility that returns `"+new"` / `"-new"` when current is 0.

**Change 3 — Correct diff colors (from Phase 3):**
- Replace `text-status-warning` (increase) → `text-status-success`.
- Replace `text-status-info` (decrease) → `text-status-error`.
- This is a breaking change from Phase 3's color usage; the Phase 4 contract supersedes it.

**Grid layout adjustment:**
- No-preset: `grid-cols-[1.2fr_100px_1.5fr]` (unchanged)
- With-preset: `grid-cols-[1.2fr_80px_140px_1.5fr]` — narrow current column, wider preset+delta column

### Modified: ParameterSection (collapsed-by-default + changes-count badge)

**Change 1 — Collapsed by default after preset selection (D-07):**
- When `selectedPresetId` changes to a non-null value, all sections collapse via `collapseAll()` in tuningWizardStore.
- Collapse is triggered in `TuningWizard.tsx`'s `handleSelect` handler when a new preset is selected (not on deselect, not on re-render).
- Individual sections remain expandable by clicking the header.

**Change 2 — Changes-count badge on collapsed section headers (Claude's discretion):**
- When `hasPreset && isCollapsed && changedCount > 0`:
  - Show badge: `{changedCount} changed` in `text-status-success` color, `bg-status-success/14` background.
  - Badge style: `rounded-full px-2 py-0.5 font-sans text-[13px]`
  - Position: inline in header row, after the existing field-count badge.
- The badge gives builders a glance-able signal for which sections to drill into.

New action `collapseAll()`:

## Interaction Contracts

### Metric Cards Appearance

1. No preset selected: MetricSectionCards is absent from DOM. Parameter browser shows all fields expanded (Phase 3 default).
2. User selects a preset: `handleSelect` fires. `selectPreset(id)` sets selected preset. `collapseAll()` collapses all sections. MetricSectionCards renders with `animate-unfurl-in` (300ms, ease-unfurl fade + scale from 0.97). Metric cards show immediately computed current vs preset values.
3. User selects a different preset: `collapseAll()` fires again. MetricCards update with new delta values. No animation re-trigger — values update in place.
4. User deselects (clicks selected preset): MetricSectionCards unmounts. Section collapse state is NOT reset — preserves whatever state the builder established.

### Tooltip Interactions

**Field label tooltips (ParameterRow):**
1. Hover over any field label: Tippy appears after 250ms delay. Shows description + optional interaction note + impact badge.
2. Mouse leaves: tooltip fades after 100ms delay.
3. Tooltip does not block row click or text selection. `interactive: false`.

**Formula tooltips (MetricRow):**
1. Hover over metric label in MetricCard: Tippy appears after 200ms delay. Shows formula copy.
2. Mouse leaves: tooltip fades after 100ms delay.

### Delta Display

- `pctDelta(oldVal: number, newVal: number): string` — extracted as a standalone utility in `src/lib/tuning/deltaUtils.ts`.
- Logic:
  - `oldVal === newVal` → return `"—"` (no delta shown)
  - `oldVal === 0` → return `newVal > 0 ? "+new" : "-new"` (guards division by zero)
  - Otherwise → `${Math.abs((newVal - oldVal) / Math.abs(oldVal) * 100).toFixed(1)}%`
- Display format in ParameterRow: `▲ {pct}` (increase) or `▼ {pct}` (decrease) appended after the numeric value.
- Display format in MetricRow: `{formattedPresetValue} ▲ {pct}` (no line-break; all inline in the preset column).

### Collapse/Expand Sections

- Clicking a collapsed section header: expands that section only.
- Clicking an expanded section header: collapses that section only.
- `collapseAll()` action: sets all 4 sections collapsed simultaneously, no animation per-section (instant state change, then sections render collapsed).

## Extracted Utilities

Two pure functions extracted to `creator/src/lib/tuning/deltaUtils.ts` for testability (Wave 0 requirement per 04-RESEARCH.md):

**`pctDelta(oldVal: number, newVal: number): string`**
- Input: current and preset numeric values
- Output: formatted string like `"▲ 12.5%"`, `"▼ 8.3%"`, `"—"`, `"+new"`, or `"-new"`
- Edge cases: `oldVal === newVal` → `"—"`; `oldVal === 0` → `"+new"` or `"-new"`

**`buildTooltipContent(meta: FieldMeta): string`**
- Input: FieldMeta with `description`, optional `interactionNote`, `impact`
- Output: HTML string for Tippy `allowHTML: true` content
- Structure: `description` paragraph + optional `interactionNote` paragraph + impact badge span
- Impact colors inline: HIGH = `color: #dbb8b8`, MEDIUM = `color: #bea873`, LOW = `color: #95a0bf`
