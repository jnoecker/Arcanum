# Phase 8 -- UI Design Contract

> Visual and interaction contract for Story Editor. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Arcanum design system) |
| Preset | not applicable |
| Component library | custom (`FormWidgets.tsx`: `DialogShell`, `ActionButton`, `Spinner`, `EditableField`) + `ConfirmDialog` |
| Icon library | inline SVG (no icon library -- project uses hand-authored SVG icons) |

Source: `ARCANUM_STYLE_GUIDE.md`, `creator/src/index.css`, Phase 7 UI-SPEC

---

## Focal Point

The horizontal SceneTimeline strip is the primary focal point of the Story Editor. It anchors the builder's spatial model of the story as a sequence of scenes, and every interaction (add, select, reorder, right-click) begins there.

---

## Spacing Scale

**Standard set:** 4, 8, 16, 24, 32, 48, 64 (multiples of 4).

**Project-level exception:** 20px (`p-5`) is included as an approved value. It is an established codebase convention already used across `ArticleBrowser`, `StoryEditorPanel`, and other panel surfaces. It appears in the standard set as the `lg` token below.

Declared values:

| Token | Value | Usage in Phase 8 |
|-------|-------|-------------------|
| xs | 4px | Inline icon gaps, badge padding-inline, gap between undo/redo buttons |
| sm | 8px | Scene card internal padding, gap between timeline cards, gap between context menu items |
| md | 16px | Default element spacing, gap between scene detail sections, timeline strip padding |
| lg | 20px | Panel padding (`p-5`), matches existing `panel-surface` usage (project exception) |
| xl | 24px | Gap between major layout sections (timeline strip vs detail editor) |
| 2xl | 32px | Top/bottom padding in zero-scenes empty state |
| 3xl | 48px | Not used in Phase 8 |

Exceptions: Timeline strip height is fixed at 120px (not a spacing token -- a layout constant).

Source: Phase 7 UI-SPEC spacing scale (inherited), existing `StoryEditorPanel.tsx` line 170

## StoryEditorPanel -- Layout Contract (Phase 8 Rewrite)

**Context:** Replaces the Phase 7 placeholder layout. Rendered inside `ArticleBrowser` when `selectedArticle.template === "story"`.

**Container:** Inherits existing `flex flex-col gap-6` wrapper from Phase 7.

### Overall Structure (vertical stack)

```
+----------------------------------------------------------+
| Section 0: Header Bar (title, zone badge, undo/redo)     |
+----------------------------------------------------------+
| Section 1: Collapsible Story Settings (cover image)      |
+----------------------------------------------------------+
| Section 2: SceneTimeline (horizontal strip, 120px tall)  |
+----------------------------------------------------------+
| Section 3: SceneDetailEditor (fills remaining height)    |
+----------------------------------------------------------+
| Section 4: Metadata Footer (timestamps, story ID)        |
+----------------------------------------------------------+
```

### Section 0: Header Bar
- Unchanged from Phase 7: `EditableField` for title, zone badge pill, undo/redo buttons
- Source: existing `StoryEditorPanel.tsx` lines 171-215

### Section 1: Collapsible Story Settings
- Collapsed by default. Toggle button: small chevron + "Story Settings" label in `text-2xs text-text-muted`
- When expanded: cover image thumbnail (if set) or dashed placeholder (240px x 160px), plus "Change Image" / "Add Cover" action button
- Purpose: Reclaims vertical space for the timeline + detail editor (CONTEXT.md D-04, Claude's discretion decision)
- Animation: Height expand/collapse, 300ms `ease-unfurl`

### Section 2: SceneTimeline (horizontal strip)
- Fixed height: 120px
- Background: `panel-surface-light` class (subtle recessed appearance)
- Border radius: `rounded-xl` (12px)
- Horizontal scroll: `overflow-x-auto` with custom scrollbar styling
- Contains: `DndContext` > `SortableContext` with `horizontalListSortingStrategy`
- Scene cards arranged in a single horizontal row with 8px gap
- Inner padding: 16px on all sides
- "Add Scene" button positioned at the end of the row (always visible as the last element in the scroll area)
- When zero scenes: centered empty state text "Add your first scene" with prominent "Add Scene" button
- Scroll indicators: Fade-out gradient overlays (24px wide) at left/right edges when content overflows. `pointer-events-none`.
- Source: CONTEXT.md D-01, D-02, D-03

### Section 3: SceneDetailEditor
- Fills remaining vertical space: `flex-1 min-h-0 overflow-y-auto`
- Contains (vertical stack, 16px gap):
  1. Scene title as `EditableField` with section heading styling
  2. Template picker (button group)
  3. Narration section: `LoreEditor` component
  4. DM Notes section: collapsible `DmNotesSection`
- When no scenes exist: hidden (Section 2 shows the zero-scenes empty state instead)
- Source: CONTEXT.md D-08, D-09, D-10, D-12

### Section 4: Metadata Footer
- Unchanged from Phase 7: created/updated timestamps, story ID
- Source: existing `StoryEditorPanel.tsx` lines 244-253

## SceneCard -- Visual Contract

**Dimensions:** 140px wide x 88px tall (fixed). `flex-shrink-0`.

**Layout (vertical stack, 8px gap):**
1. Top row: scene index number (`text-2xs text-text-muted`) right-aligned
2. Middle: scene title (single line, `truncate`, `text-2xs text-text-primary`)
3. Bottom: template badge (if set) or empty space

**States:**

| State | Visual |
|-------|--------|
| Default | `border border-border-default bg-bg-primary rounded-xl`, cursor `grab` |
| Hover | `bg-bg-tertiary`, border unchanged |
| Selected | `border-accent bg-bg-elevated`, subtle glow: `shadow-[0_0_12px_rgba(168,151,210,0.2)]` |
| Dragging | `opacity-50`, card at original position shows reduced opacity. `DragOverlay` renders a full-opacity clone at cursor position |
| Drag over (drop target) | Neighboring cards shift with 200ms CSS transition to make space |

**Right-click:** Opens `SceneContextMenu` at cursor position.

**Click:** Selects the scene (`setActiveScene(scene.id)`). Does not conflict with drag due to `PointerSensor` distance constraint of 5px.

Source: CONTEXT.md D-05, D-06, D-07, RESEARCH.md Pattern 2

## Scroll Indicators -- Visual Contract

**When timeline content overflows horizontally:**

- Left indicator: 24px wide gradient from `bg-bg-primary` to `transparent`, positioned at the left edge of the timeline strip. `pointer-events-none`, `z-[1]`.
- Right indicator: Same, mirrored. Gradient from `transparent` to `bg-bg-primary`.
- Visibility: Left indicator hidden when scrolled to the start. Right indicator hidden when scrolled to the end. Toggled via scroll position check.
- No arrow buttons -- scroll affordance is purely visual via gradients + native horizontal scroll (touchpad, shift+scroll, or drag-scrollbar).

Source: CONTEXT.md D-02, Claude's discretion

## Interaction States

### SceneTimeline

| State | Behavior |
|-------|----------|
| Zero scenes | Centered empty state with prominent "Add your first scene" CTA |
| 1+ scenes | Horizontal row of scene cards + "Add Scene" pill at end |
| Overflowing | Horizontal scroll enabled, gradient fade indicators at edges |
| Dragging a card | DragOverlay shows card clone at cursor. Original card at 50% opacity. Other cards animate aside (200ms CSS transition) |
| After drag completes | `reorderScenes` called with new order. `activeSceneId` unchanged (keyed on ID, not index) |
| After "Add Scene" click | New scene appended, auto-selected, timeline scrolls to end |
| Right-click on card | Context menu appears at cursor position |

### SceneDetailEditor

| State | Behavior |
|-------|----------|
| No scene selected | Hidden (always a scene selected when scenes exist per D-03) |
| Scene selected | Renders: scene title, template picker, narration editor, DM notes section |
| Narration editing | LoreEditor manages TipTap state internally. `onCommit` updates store |
| DM notes editing | Plain textarea. `onChange` updates store via `updateScene` |
| Template picker: click template | If scene has content: show ConfirmDialog. If empty: apply silently |
| Template picker: click "Clear Template" | Clears `scene.template` field only. Does not clear title/narration |
| Switching selected scene | Detail editor re-renders with new scene data. LoreEditor receives new `value` |

### Context Menu

| State | Behavior |
|-------|----------|
| Open | Positioned at cursor. Focus trapped within |
| Item hover | Background highlight `bg-bg-hover` |
| "Duplicate Scene" | Creates copy of scene with new ID, inserted after original, auto-selected |
| "Delete Scene" | Shows ConfirmDialog. On confirm: removes scene, selects adjacent scene (or none if last) |
| "Apply Template" > submenu | Submenu appears to the right. If scene has content: ConfirmDialog gates it. If empty: applies silently |
| "Clear Template" | Clears `scene.template` field. No confirmation needed |
| Escape or click outside | Menu dismisses |

### Auto-Selection Logic (CONTEXT.md D-03)

| Event | Selection Behavior |
|-------|--------------------|
| Story opens with scenes | First scene (sortOrder 0) auto-selected |
| First scene added to empty story | New scene auto-selected |
| Scene deleted | Select the next scene in order; if last scene was deleted, select previous; if all deleted, no selection |
| Scene duplicated | Duplicated (new) scene auto-selected |
| Scene reordered | Selection stays on the same scene (unchanged -- keyed by ID) |

## Animation

| Element | Animation | Timing | Easing |
|---------|-----------|--------|--------|
| Scene card drag | `DragOverlay` follows cursor; original card fades to 50% opacity | Immediate | linear (pointer tracking) |
| Neighbor cards shift during drag | `translateX` shift to make space for dragged card | 200ms | `ease-cosmic` (via dnd-kit CSS.Transform) |
| Scene card appear (new scene added) | `animate-unfurl-in` (fade + translateY from 6px) | 400ms | `ease-unfurl` |
| Context menu appear | Fade in + scale from 0.95 | 150ms | `ease-unfurl` |
| Context menu dismiss | Fade out | 100ms | `ease-dissolve` |
| Story Settings collapse/expand | Height animate | 300ms | `ease-unfurl` |
| DM Notes collapse/expand | Height animate | 200ms | `ease-unfurl` |
| Template badge appear (on apply) | `animate-unfurl-in` | 400ms | `ease-unfurl` |
| Scroll fade indicators | Opacity toggle based on scroll position | 150ms | `ease-cosmic` |
| Selected card border transition | Border color change | 160ms | `ease-cosmic` |

Note: Phase 8 does NOT use Motion (framer-motion). All animations are CSS transitions and the existing `@keyframes` in `index.css`, plus dnd-kit's built-in CSS transform transitions. Motion is introduced in Phase 10.

Source: `ARCANUM_STYLE_GUIDE.md` Motion section, existing animation keyframes in `index.css`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard drag-and-drop | dnd-kit `KeyboardSensor` enables Tab to focus cards, Space to pick up, Arrow keys to move, Space to drop |
| Screen reader drag announcements | dnd-kit provides built-in ARIA live region announcements for drag operations |
| Context menu keyboard | Escape to dismiss, Arrow keys to navigate items, Enter to select |
| Context menu focus trap | Focus moves into context menu on open, returns to triggering card on close |
| Form labels | "Narration" section heading associated with LoreEditor via `aria-labelledby`. "DM Only" label on notes section. |
| Collapsible sections | `aria-expanded` on toggle buttons for Story Settings and DM Notes. Collapsed content uses `aria-hidden`. |
| Template picker | Each template button has `aria-pressed` matching active state |
| Scene card selection | `aria-selected` on the selected card within the timeline |
| ConfirmDialog | Inherits focus trap and `role="dialog"` from existing `ConfirmDialog` + `DialogShell` |
| Color contrast | Badge text on badge backgrounds: `#8caec9` on `rgba(140,174,201,0.18)` over `#2a3149` base -- effective contrast exceeds 4.5:1 |
| Drag handle visual | `cursor-grab` on scene cards communicates draggability. `cursor-grabbing` during active drag. |

Source: dnd-kit accessibility docs, existing `useFocusTrap` pattern
