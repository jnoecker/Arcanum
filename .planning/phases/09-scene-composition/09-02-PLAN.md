**Implementation details:**

1. **Positioning:** Use `resolveEntityPosition(entity)` from sceneLayout.ts to get `{ x, y }` percentages. Render as `position: absolute` with `left: {x}%`, `top: {y}%`, `transform: translate(-50%, -100%)` (anchor at bottom-center of sprite, per UI-SPEC).

2. **Scale/depth:** Use `getEntityScale(entity)` for the width. Front-row: 72px width. Back-row: 56px width (72 * 0.78 = ~56). Use `isBackRow(entity.slot)` to determine if back-row opacity (90%) applies. These values come from the UI-SPEC EntityOverlay table.

3. **Image loading:** Use `useImageSrc(entityImage)` for the entity sprite. When `src` is null or entityImage is undefined, render a placeholder: rounded rectangle at the sprite's size, `bg-bg-elevated/80`, with an inline SVG icon centered (skull for mob, gem for item, user for npc) in `text-text-muted`. The icons should be simple 20px inline SVGs.

4. **Name label:** Below the sprite, render entity name in `font-body text-xs text-white` (Crimson Pro 12px) centered, with `text-shadow: 0 1px 3px rgba(0,0,0,0.8)` for readability over any background (per UI-SPEC typography rules).

5. **Drag interaction (per D-08, RESEARCH Pitfall 5):**
   - `onPointerDown`: Call `e.currentTarget.setPointerCapture(e.pointerId)`. Record initial offset (pointer position relative to entity center) in a `useRef`. Set local dragging state via `useState(false)`.
   - `onPointerMove`: Only process if `e.currentTarget.hasPointerCapture(e.pointerId)`. Calculate new percentage position from pointer coordinates relative to `containerRef.current.getBoundingClientRect()`. Use `clampPosition` from sceneLayout.ts. Update LOCAL `useState` position (not the store -- per Pitfall 5).
   - `onPointerUp`: Release pointer capture. Call `onReposition(entity.id, localPosition)` to commit to store. Reset dragging state.
   - During drag: add `cursor-grabbing` class, reduce opacity to 85%.

6. **Selection (per D-04, UI-SPEC):**
   - Click (without drag -- if pointer didn't move more than 3px) calls `onSelect(entity.id)`.
   - When `selected` is true: show `ring-2 ring-accent/45` (2px accent ring).
   - On hover (not selected): show `ring-2 ring-accent/30` with 180ms transition.

7. **Removal (per D-04, UI-SPEC):**
   - On hover, show an X button at top-right of the entity: 16px circle, `bg-black/60 hover:bg-black/80`, white X icon (8px inline SVG). Clicking calls `onRemove(entity.id)`.
   - Delete key handling is managed by the parent ScenePreview component (not here).

8. **Accessibility:** Add `role="button"`, `aria-label="{entityName}"`, `aria-pressed={selected}`, `aria-roledescription="draggable entity"`, `tabIndex={0}`.

9. **CSS classes on the wrapper div:**
   - `cursor-grab` default, `cursor-grabbing` when dragging
   - `transition-shadow duration-[180ms]` for hover/selection ring transitions
   - `pointer-events-auto` (since ScenePreview may have pointer-events-none on some layers)

   - Render: `<img src={roomSrc} alt="" className="absolute inset-0 h-full w-full object-cover" />` when roomSrc is available.

3. **Entity resolution:**
   - Read the scene's `entities` array (default to empty `[]`).
   - For each entity, look up its name and image from zoneStore based on `entityType` and `entityId`:
     - "mob": `zoneState?.data.mobs?.[entity.entityId]` -> name from `.name`, image from `.image`
     - "item": `zoneState?.data.items?.[entity.entityId]` -> name from `.displayName`, image from `.image`
     - "npc": same as mob (fallback)
   - Create a helper function `resolveEntityInfo(entity: SceneEntity, zoneState: ZoneState | undefined)` local to this file that returns `{ name: string; image?: string }`.

4. **Entity layers (Layers 1-2, per D-13, UI-SPEC):**
   - Separate entities into back-row and front-row using `isBackRow(entity.slot)`.

     - `selected` (from local `selectedEntityId` state)

5. **Selection state:**

   - When an entity is selected, listen for Delete key press to remove it:

     useEffect(() => {
       if (!selectedEntityId) return;

         if (e.key === "Delete" || e.key === "Backspace") {

       window.addEventListener("keydown", handleKeyDown);
       return () => window.removeEventListener("keydown", handleKeyDown);
