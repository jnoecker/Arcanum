## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, entity label gap to sprite |
| sm | 8px | Compact element spacing, picker row padding, tab gaps |
| md | 16px | Default element spacing, preview-to-narration gap, section gaps within picker |
| lg | 24px | Section padding, preview container internal padding for narration overlay |
| xl | 32px | Layout gaps between main content and entity picker sidebar |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Entity sprite render size: 72px front-row, 56px back-row (not spacing tokens -- these are entity dimensions)
- Entity name label: 4px below sprite bottom edge
- 16:9 preview minimum height: 320px (derived from container width, not a spacing token)

Source: Default 8-point scale, entity sizes from CONTEXT.md D-09 (64-80px range -- using 72px front, 56px back for clean depth ratio)

## Interaction Contracts

### Entity Picker Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click room row | Sets `scene.roomId`, room bg appears in preview | Row gets `bg-accent/14` + 2px left accent bar; previous active room loses highlight |
| Click mob/item row | Adds `SceneEntity` to scene, entity appears in preview at next available preset slot | Brief flash of `bg-accent/14` on row (200ms), entity fades in at slot position (180ms unfurl) |
| Type in search | Filters entity list by name (case-insensitive substring match) | Instant filter, no debounce needed for local data |
| Switch tab | Shows entities of that type | Active tab gets accent underline, content area transitions (no animation -- instant swap) |
| Collapse picker | Sidebar width animates to 32px strip | 200ms transition with `ease-unfurl`, strip shows vertical entity-picker icon |
| Expand picker | Sidebar width animates to 280px | 200ms transition with `ease-unfurl` |

### Scene Preview Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click entity in preview | Selects entity (for repositioning or deletion) | Selected ring appears (2px accent/45), other entities lose selection |
| Click empty area in preview | Deselects any selected entity | Selected ring removed |
| Pointer down + drag on entity | Begins free-form repositioning | Entity follows pointer with `cursor-grabbing`, slight opacity reduction to 85% |
| Pointer up after drag | Commits position to `storyStore.updateScene` as percentage `{ x, y }`, clears slot | Entity snaps to final position, opacity restores to 100% |
| Hover entity | Shows grab cursor and hover ring | `ring-accent/30` ring, `cursor-grab` |
| Hover entity X button | Shows delete affordance | X button background brightens to `bg-black/80` |
| Click entity X button | Removes entity from scene | Entity fades out (120ms dissolve), slot becomes available |
| Press Delete key (entity selected) | Removes selected entity from scene | Same fade-out as X button |
| Hover empty preview | No special behavior | Standard cursor |

### State Management Contracts

| State Change | Store | Method | Undo Behavior |
|-------------|-------|--------|---------------|
| Set room background | storyStore | `updateScene(storyId, sceneId, { roomId })` | Single undo snapshot |
| Add entity | storyStore | `updateScene(storyId, sceneId, { entities: [...existing, newEntity] })` | Single undo snapshot |
| Remove entity | storyStore | `updateScene(storyId, sceneId, { entities: filtered })` | Single undo snapshot |
| Reposition entity (drag) | local `useState` during drag, then storyStore on drop | `updateScene(storyId, sceneId, { entities: updated })` | One snapshot per drag operation (not per frame) |

Source: CONTEXT.md D-01 through D-13, RESEARCH.md Patterns 1-4, Pitfall 5 (drag snapshots)

## Animation Contracts

All animations follow Arcanum motion principles: unhurried, no snapping, use `--ease-unfurl` and `--ease-cosmic` easings.

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Entity fade-in (added to preview) | 180ms | `ease-unfurl` | Entity added to scene |
| Entity fade-out (removed from preview) | 120ms | `ease-dissolve` | Entity removed from scene |
| Picker sidebar collapse/expand | 200ms | `ease-unfurl` | Collapse toggle clicked |
| Room background crossfade | 300ms | `ease-cosmic` | Room selection changed |
| Entity hover ring appear | 180ms | `ease-cosmic` | Pointer enters entity |
| Entity selection ring appear | 120ms | `ease-cosmic` | Entity clicked |
| Tab active indicator slide | 180ms | `ease-cosmic` | Tab switched |

No Motion library needed for Phase 9 -- all animations are CSS transitions using existing easing custom properties.

Source: ARCANUM_STYLE_GUIDE.md Motion & Animation section, existing transition patterns in `DmNotesSection.tsx`

## Accessibility Contracts

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation in picker | Tab through entity rows, Enter to add to scene, arrow keys within tab list |
| Entity picker tabs | `role="tablist"` + `role="tab"` + `role="tabpanel"`, `aria-selected` on active tab |
| Entity selection in preview | `role="button"` on each entity overlay, `aria-label="{entity name}"`, `aria-pressed` when selected |
| Delete key binding | Only fires when an entity is selected (not globally) |
| Drag operation | `aria-roledescription="draggable entity"`, `aria-label="Drag to reposition {entity name}"` |
| Collapse toggle | `aria-expanded`, `aria-controls` targeting the picker content |
| Search input | `aria-label="Filter {rooms/mobs/items}"`, `role="searchbox"` |
| Preview empty state | `role="status"` with screen reader text |
| Color contrast | All text meets WCAG AA against its background: white text on black/60 gradient (narration), `text-text-primary` on `bg-bg-secondary` (picker) |

Source: Default accessibility best practices, existing `DmNotesSection.tsx` aria patterns

Key layout rules:
- Outer container: `flex gap-4 flex-1 min-h-0` (horizontal split)
- Main content: `flex flex-col gap-4 flex-1 min-h-0 overflow-y-auto`

- When picker is collapsed: picker becomes `w-8` vertical strip
- ScenePreview: `w-full aspect-video` within the main content column
- All containers respect `min-h-0 flex-1` for proper flex scrolling (CLAUDE.md pitfall)
