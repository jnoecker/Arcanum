## Component Inventory

New components for this phase and their visual contracts:

### CinematicRenderer (animation orchestrator)

| Property | Value |
|----------|-------|
| Container | `relative aspect-video w-full overflow-hidden rounded-lg bg-black` |
| Content | `AnimatePresence` wrapping `CinematicScene` keyed by `scene.id` |
| LazyMotion boundary | Wraps entire component tree with `LazyMotion features={loadMotionFeatures} strict` |
| AnimatePresence mode | `"sync"` for crossfade, `"wait"` for fade-to-black (dynamic per scene's `transition.type`) |
| Props | `scenes: Scene[]`, `currentIndex: number`, `playing: boolean`, `onComplete?: () => void`, `onSceneChange?: (index: number) => void`, `narrationSpeed?: NarrationSpeed` |
| Tauri deps | NONE -- fully portable component |
| Image resolution | Scenes arrive with images already resolved to URLs/data-URLs via props (no store access inside) |

Source: CONTEXT.md D-10, D-11, RESEARCH.md Pattern 1

### CinematicScene (single animated scene)

| Property | Value |
|----------|-------|
| Container | `m.div` with `absolute inset-0` |
| Entry animation | `initial={{ opacity: 0 }}`, `animate={{ opacity: 1 }}` |
| Exit animation | `exit={{ opacity: 0 }}` |
| Transition duration (crossfade) | 500ms (`transition={{ duration: 0.5 }}`) |
| Transition duration (fade-to-black) | 300ms each direction (`transition={{ duration: 0.3 }}`) |
| Easing | `[0.4, 0, 0.2, 1]` (ease-cosmic) for scene transitions |
| Layer order | Same as ScenePreview: bg (z-0), back-row entities (z-10), front-row entities (z-20), narration (z-30) |
| Animation sequence | 1. Scene fades in, 2. Entity entrance animations play concurrently, 3. Narration typewriter begins after entity entrances (200ms delay), 4. Scene holds until advance, 5. Entity exits trigger, 6. ~200ms later scene transition begins |

Source: CONTEXT.md D-10, RESEARCH.md Pattern 2, Open Question 2

### AnimatedEntity (entity with entrance/exit paths)

| Property | Value |
|----------|-------|
| Container | `m.div` with `absolute` positioning, `left: {x}%`, `top: {y}%`, `transform: translate(-50%, -100%)` |
| Entrance (with path) | CSS `offset-path: path("...")` + Motion animates `offsetDistance: "0%" -> "100%"` with concurrent `opacity: 0 -> 1` |
| Entrance (fade-in-place) | No offset-path, just `opacity: 0 -> 1` with slight `y: 4 -> 0` movement |
| Exit (with path) | CSS `offset-path: path("...")` + Motion animates `offsetDistance: "0%" -> "100%"` with concurrent `opacity: 1 -> 0` |
| Exit (fade-out) | `opacity: 1 -> 0` over 600ms |
| Entrance easing | `[0.16, 1, 0.3, 1]` (ease-unfurl) -- gentle start, decisive arrival |
| Exit easing | `[0.7, 0, 0.84, 0]` (ease-dissolve) -- starts moving, then dissolves |
| Entrance default duration | 1.0s (per preset, range 0.8-1.5s) |
| Exit default duration | 0.8s (per preset, range 0.6-0.8s) |
| Reduced motion | Skip path animation, show entity at final position instantly, opacity transition only (200ms) |
| Child content | Entity sprite image or placeholder (reuse `EntityOverlay` visual rendering without drag interaction) |

Source: CONTEXT.md D-01, D-02, RESEARCH.md Pattern 3, Code Examples (presets)

## Animation Contracts

All animations use Motion library (`m` component inside `LazyMotion` boundary). Easing curves reference Arcanum style guide.

### Scene Transitions

| Animation | Duration | Easing | Mode | Trigger |
|-----------|----------|--------|------|---------|
| Crossfade (scene A -> B) | 500ms overlap | `[0.4, 0, 0.2, 1]` (ease-cosmic) | `AnimatePresence mode="sync"` | Scene index advances |
| Fade-to-black (scene A out) | 300ms | `[0.7, 0, 0.84, 0]` (ease-dissolve) | `AnimatePresence mode="wait"` | Scene index advances |
| Fade-to-black (scene B in) | 300ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) | `AnimatePresence mode="wait"` | After exit completes |
| Fade-to-black (black hold) | ~200ms | N/A (natural gap) | N/A | Between exit complete and enter start |

### Entity Movement

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Enter from left | 1000ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) | Scene starts playing |
| Enter from right | 1000ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) | Scene starts playing |
| Enter from bottom | 1200ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) | Scene starts playing |
| Rise from shadows | 1500ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) | Scene starts playing |
| Fade in place | 800ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) | Scene starts playing |
| Exit stage left | 800ms | `[0.7, 0, 0.84, 0]` (ease-dissolve) | Scene exit triggered (~200ms before transition) |
| Exit stage right | 800ms | `[0.7, 0, 0.84, 0]` (ease-dissolve) | Scene exit triggered |
| Fade out | 600ms | `[0.7, 0, 0.84, 0]` (ease-dissolve) | Scene exit triggered |

Entity entrance opacity: concurrent `opacity 0 -> 1` over same duration as path movement.
Entity exit opacity: concurrent `opacity 1 -> 0` over same duration as path movement.
All entities in a scene start their entrance animations simultaneously (no stagger between entities).

### Narration Typewriter

| Speed | Word Duration | Word Gap (stagger) | Easing |
|-------|--------------|-------------------|--------|
| Slow | 200ms | 120ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) |
| Normal | 150ms | 80ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) |
| Fast | 100ms | 50ms | `[0.16, 1, 0.3, 1]` (ease-unfurl) |

Narration starts 200ms after entity entrance animations begin (not after they complete -- concurrent with a slight delay for visual hierarchy).

### Scene Playback Sequence (orchestration timeline)

```
t=0ms       Entity entrance animations begin (all entities concurrent)
t=200ms     Narration typewriter begins
t=~1200ms   Entity entrances complete (longest preset: Rise from shadows at 1500ms)
t=varies    Narration typewriter completes (depends on word count and speed)
t=hold      Scene holds at final state until advance trigger
t=advance   Entity exit animations begin
t=advance+200ms  Scene transition begins (crossfade or fade-to-black)
t=advance+~1000ms  Transition complete, next scene plays
```

### Editor-Specific Animations (non-cinematic)

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Preview playback button hover glow | 180ms | `ease-cosmic` | Pointer enters button |
| Playing state warm pulse | 2000ms loop | `ease-cosmic` | During playback |
| Dropdown open | 150ms | `ease-unfurl` | Click dropdown trigger |
| Dropdown close | 100ms | `ease-dissolve` | Click outside or select option |
| Segmented control active slide | 180ms | `ease-cosmic` | Click speed segment |

Source: ARCANUM_STYLE_GUIDE.md Motion & Animation, CONTEXT.md D-04 through D-09, RESEARCH.md patterns 1-6

## Interaction Contracts

### CinematicRenderer Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Scene advance (external trigger) | Plays exit animations on current scene, then transitions to next scene | Entity exits + crossfade/fade-to-black |
| Scene retreat (external trigger) | Transitions to previous scene (no exit animations -- instant crossfade back) | Crossfade to previous scene |
| Playing state set to true | Begins animation sequence: entrances, then narration | Entities animate in, words reveal sequentially |
| Playing state set to false | Animations pause at current position | All elements freeze at current state |
| All animations complete | Fires `onComplete` callback | Scene holds at final static state |
| Rapid scene advancement | AnimatePresence handles overlap; previous exit may not complete | Short transition durations prevent visible pile-up |

### Preview Playback Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click "Preview" button | Plays current scene's entrance + narration animations inline | Button changes to "Stop" with warm accent; entities animate in, narration reveals |
| Click "Stop" button | Stops all animations, resets to static view | Button reverts to "Preview" with accent color; entities snap to final positions, narration fully visible |
| Scene changes while previewing | Stops preview, resets to new scene static view | Button reverts to idle state |

### Transition Dropdown Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click dropdown trigger | Opens dropdown menu below trigger | Menu appears with 150ms ease-unfurl |
| Hover option | Highlights option | `bg-bg-hover` with 180ms transition |
| Click option | Selects transition type, saves to `scene.transition`, closes dropdown | Selected option gets `text-accent` + checkmark, dropdown closes 100ms |
| Click outside | Closes dropdown without change | 100ms ease-dissolve |

### Path Preset Picker Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click entrance dropdown | Opens entrance preset list | Same dropdown styling as TransitionDropdown |
| Select entrance preset | Saves preset ID to `entity.entrancePath` via storyStore | Checkmark appears on selected option |
| Click exit dropdown | Opens exit preset list | Same dropdown styling |
| Select exit preset | Saves preset ID to `entity.exitPath` via storyStore | Checkmark appears on selected option |
| Select "None" | Clears the path (no animation for that direction) | "None" shown in italic muted text |

### Narration Speed Selector Interactions

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click speed segment | Sets `scene.narrationSpeed` override | Active segment slides with 180ms transition, `bg-bg-elevated text-accent` |
| Speed matches story default | No override indicator | Clean segmented control |
| Speed differs from story default | Override is active | 4px accent dot appears next to "Narration Speed" label |

### Reduced Motion Behavior

| Context | Behavior |
|---------|----------|
| `prefers-reduced-motion: reduce` detected | All animations run at 0ms duration (instant final state) |
| Scene transitions | Instant swap (no crossfade or fade-to-black) |
| Entity movement paths | Entity appears at final position immediately |
| Typewriter narration | All words visible immediately (no stagger) |
| Preview playback button | Still functional, label shows "Preview (reduced motion)" |
| Detection method | `window.matchMedia("(prefers-reduced-motion: reduce)").matches` (matches existing pattern in `SpringPanel.tsx`, `Starfield.tsx`) |

### State Management Contracts

| State Change | Store | Method | Undo Behavior |
|-------------|-------|--------|---------------|
| Set transition type | storyStore | `updateScene(storyId, sceneId, { transition: { type } })` | Single undo snapshot |
| Set entity entrance path | storyStore | `updateScene(storyId, sceneId, { entities: updated })` | Single undo snapshot |
| Set entity exit path | storyStore | `updateScene(storyId, sceneId, { entities: updated })` | Single undo snapshot |
| Set narration speed | storyStore | `updateScene(storyId, sceneId, { narrationSpeed })` | Single undo snapshot |
| Playback state (playing/paused) | Local React state in PreviewPlayback | `useState<boolean>` | Not undoable (ephemeral UI state) |
| Current scene index | External prop (CinematicRenderer receives it) | Controlled by parent (StoryEditorPanel or presentation hook) | Not undoable |

Source: CONTEXT.md D-01 through D-12, RESEARCH.md Architecture Patterns, Phase 9 state patterns

## Accessibility Contracts

| Requirement | Implementation |
|-------------|----------------|
| Reduced motion | Respect `prefers-reduced-motion: reduce` -- skip all animations, show final states instantly |
| Preview playback button | `aria-label="Preview scene animation"` (idle) / `aria-label="Stop scene animation"` (playing), `role="button"` |
| Transition dropdown | `role="listbox"`, `aria-label="Scene transition type"`, options with `role="option"` and `aria-selected` |
| Path preset dropdowns | `role="listbox"`, `aria-label="Entity entrance path"` / `aria-label="Entity exit path"` |
| Narration speed selector | `role="radiogroup"`, `aria-label="Narration speed"`, segments with `role="radio"` and `aria-checked` |
| Typewriter narration | `aria-live="polite"` on the narration container so screen readers announce the full text (not word-by-word) |
| CinematicRenderer | `role="region"`, `aria-label="Scene playback"`, `aria-live="polite"` for scene change announcements |
| Scene transition announcement | Visually hidden `<span>` with `aria-live="assertive"`: "Scene {n} of {total}: {scene title}" on scene change |
| Keyboard in CinematicRenderer | Not handled in this phase -- Phase 11 (Presentation Mode) adds keyboard navigation |
| Color contrast | White narration text on black/60 gradient: exceeds 7:1 ratio (WCAG AAA). Accent text (#a897d2) on elevated bg (#313a56): 3.6:1 (meets AA for UI components) |
