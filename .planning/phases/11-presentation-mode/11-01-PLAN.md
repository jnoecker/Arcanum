```typescript
function resolveEntityInfo(
  entity: SceneEntity,
  zoneState: ZoneState | undefined,
): { name: string; image?: string } {
  if (!zoneState) return { name: entity.entityId };
  if (entity.entityType === "mob" || entity.entityType === "npc") {
    const mob = zoneState.data.mobs?.[entity.entityId];
    if (mob) return { name: mob.name, image: mob.image };
    return { name: entity.entityId };
  }
  if (entity.entityType === "item") {
    const item = zoneState.data.items?.[entity.entityId];
    if (item) return { name: item.displayName, image: item.image };
    return { name: entity.entityId };
  }
  return { name: entity.entityId };
}
```

**1. Create `creator/src/lib/useResolvedSceneData.ts`:**

A hook that resolves all scene images (room backgrounds + entity sprites) for a batch of scenes, producing the `resolvedSceneData` array that CinematicRenderer expects.

Since `useImageSrc` is a hook and cannot be called in a loop, this hook must use `invoke("read_image_data_url", { path })` directly in an effect, resolving all images imperatively.

```typescript
import { useState, useEffect, useMemo } from "react";
import { invoke } from "@tauri-apps/api/core";
import { useZoneStore } from "@/stores/zoneStore";
import { useAssetStore } from "@/stores/assetStore";
import { useProjectStore } from "@/stores/projectStore";
import { isR2HashPath, isLegacyImagePath } from "@/lib/useImageSrc";
import type { Scene, SceneEntity } from "@/types/story";
```

The hook signature:
```typescript
export function useResolvedSceneData(
  scenes: Scene[],
  zoneId: string,
): Array<{
  sceneId: string;
  roomImageSrc?: string;
  entities: Array<{ entity: SceneEntity; name: string; imageSrc?: string }>;
}>
```

Implementation approach:
- Read `zones` from `useZoneStore`, `assetsDir` from `useAssetStore`, `mudDir` from `useProjectStore`.
- Use `useMemo` to compute a "resolution plan" -- for each scene, determine which file paths need resolving (room image path, each entity's image path). Use the same `resolveEntityInfo` pattern from ScenePreview (inline in this hook) to get entity name and image filename from zone data.
- Use `useEffect` to invoke `read_image_data_url` for each unique file path (deduplicate paths to avoid redundant IPC calls). Build a `Map<string, string>` of `filePath -> dataUrl`. Use the same candidate path logic from `useImageSrc` (R2 hash -> assetsDir, legacy -> mudDir paths, absolute -> direct).
- Store the resolved map in state. Produce the final array by mapping scenes + their entities through the resolved map.
- Include a cleanup flag (`let cancelled = false`) in the effect to handle unmount during async resolution.

**Props interface:**

```typescript
  zoneId: string;            // For image resolution
  narrationSpeed?: NarrationSpeed; // Story-level default
```

**Component state (all local, per UI-SPEC State Management Contract):**
- `currentIndex: number` -- initialized from `initialSceneIndex`
- `playing: boolean` -- whether current scene animations are playing
- `dmNotesVisible: boolean` -- DM notes overlay toggle, default `false`
- `hudVisible: boolean` -- HUD auto-fade state, default `true`
- `cursorVisible: boolean` -- cursor auto-hide state, default `true`

**Fullscreen lifecycle (per D-04):**
- On mount: call `getCurrentWindow().setFullscreen(true)`. Wrap in try/catch (fallback: continue without fullscreen if API fails, e.g. dev mode without Tauri).
- On unmount: call `getCurrentWindow().setFullscreen(false)` in cleanup. Same try/catch pattern.

**Keyboard handler (per D-11):**
Register a `keydown` event listener on `window` in a `useEffect`. The handler:
- `ArrowRight`, `Space`, `Enter`: If `currentIndex < scenes.length - 1`, set `currentIndex` to `currentIndex + 1` and set `playing = true` (D-13: forward always replays animations).
- `ArrowLeft`: If `currentIndex > 0`, set `currentIndex` to `currentIndex - 1` and set `playing = false` (D-12: backward shows final state).
- `Escape`, `F5`: Call `handleExit()`.
- `d`, `D`, `n`, `N`: Toggle `dmNotesVisible`, but ONLY if `scenes[currentIndex].dmNotes` is a non-empty string (D-07).
- Call `e.preventDefault()` for all handled keys.
- Reset HUD timer on any keypress (set `hudVisible = true`, restart 3s timer).

**Mouse handler (per D-10):**
Register `click` and `contextmenu` event listeners on the presentation container `<div>`:
- `click` (left): Same as ArrowRight (advance). 
- `contextmenu` (right): Call `e.preventDefault()` to suppress context menu, then same as ArrowLeft (go back).
- Register `mousemove` on `window`: Reset cursor timer (set `cursorVisible = true`, restart 3s timer). Also reset HUD timer.

**HUD auto-hide timer (per D-08):**
Use a `useRef` for a timeout ID. On any keypress or mouse move, clear the timeout, set `hudVisible = true`, start new 3-second timeout that sets `hudVisible = false`. Initialize timer on mount.

**Cursor auto-hide:**
Same pattern as HUD timer but for cursor. Use a separate `useRef` timeout. Only `mousemove` resets it (keypresses do NOT restore cursor per UI-SPEC). Apply `cursor: none` style to the container div when `cursorVisible === false`.

**Exit handler:**
```typescript
const handleExit = useCallback(() => {
  const currentScene = scenes[currentIndex];
  if (currentScene) {
    onExit(currentScene.id);
  }
}, [scenes, currentIndex, onExit]);
```

**Image resolution:**
Call `useResolvedSceneData(scenes, zoneId)` to get the resolved scene data array. Pass it to CinematicRenderer.

**Reduced motion:**
Check `window.matchMedia("(prefers-reduced-motion: reduce)").matches` at module level (same pattern as PreviewPlayback.tsx). If active, pass `playing={false}` to CinematicRenderer regardless of direction (entities appear in final positions, narration fully revealed, crossfade becomes instant cut -- all handled by CinematicRenderer/CinematicScene internally).

**Render structure:**
```tsx

  className="fixed inset-0 z-[60] flex items-center justify-center bg-black"

      currentIndex={currentIndex}
```

**onComplete handler:**

**Focus management (per UI-SPEC Accessibility):**

**Cleanup:**
All event listeners, timers, and fullscreen state are cleaned up on unmount via useEffect return functions.
