| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | HUD element internal padding, scene counter padding |
| md | 16px | DM notes bar padding, HUD margin from screen edges |
| lg | 24px | DM notes bar vertical padding |
| xl | 32px | HUD inset from viewport edges |

Exceptions: The "Present" button in StoryEditorPanel header follows existing `action-button` sizing (min-height 2.75rem / 44px). Presentation mode is fullscreen with viewport-edge-relative positioning, so spacing is primarily `xl` (32px) insets from screen edges for all HUD elements.

## Layout Contract

### Fullscreen Presentation Layout

```
+------------------------------------------------------------------+
|                                                                    |
|   (pure black backdrop fills entire viewport)                      |
|                                                                    |
|   +----------------------------------------------------------+    |
|   |                                                            |    |
|   |              CinematicRenderer                             |    |
|   |              (16:9, centered, max-width to fill)           |    |
|   |                                                            |    |
|   |                                                     [3/8]  |  <-- HUD: scene counter (auto-fades)
|   |                                                            |    |
|   +----------------------------------------------------------+    |
|                                                                    |
|   +----------------------------------------------------------+    |
|   | DM Notes: "Remember to describe the sound of..."          |  <-- DM notes bar (toggled, hidden by default)
|   +----------------------------------------------------------+    |
|                                                                    |
+------------------------------------------------------------------+
```

- **Viewport fill**: `position: fixed; inset: 0; z-index: 60` (above dialog-overlay's z-50)
- **Black backdrop**: `bg-black` on the fixed container
- **CinematicRenderer sizing**: Remove `rounded-lg` border radius from presentation context. Use `w-full max-h-full` within a centered flex container, preserving 16:9 aspect ratio via `aspect-video`. The renderer scales to fill the viewport width while maintaining aspect ratio. On ultra-wide monitors, the renderer will be letterboxed (black bars top/bottom).
- **Scene counter**: Positioned `absolute top-8 right-8` inside the presentation container. Pill shape with `backdrop-blur-sm`.
- **DM notes bar**: Positioned `absolute bottom-0 left-0 right-0` inside the presentation container. Full-width bar, content max-width 960px centered.

### "Present" Button in StoryEditorPanel

Placed in the header bar between the zone name badge and the undo/redo controls:

```
[ Title ][zone-badge]          [Present] [undo][redo]
```

Button uses `action-button-primary` variant (warm-gold, pill shape) with a play-triangle SVG icon + "Present" label. Disabled when story has 0 scenes.

## Interaction Contract

### Entry Flow (D-01, D-02, D-04)

| Trigger | Behavior |
|---------|----------|
| Click "Present" button | Enter presentation from current `activeSceneId`. If no activeSceneId, start from first scene (by sortOrder). |
| Press F5 (in StoryEditorPanel context) | Same as clicking "Present". Only active when a story editor tab is focused. |
| Transition | Tauri `invoke("plugin:window|set_fullscreen", { value: true })` on the current window. Presentation component mounts after fullscreen confirmed. No fade transition -- instant. |

### Navigation (D-10, D-11, D-12, D-13)

| Input | Action |
|-------|--------|
| Right Arrow / Space / Enter | Advance to next scene. Replays entrance animations and typewriter narration. |
| Left Arrow | Go back to previous scene. Shows scene in final state (no animation replay). |
| Left click (anywhere) | Advance to next scene (same as Right Arrow). |
| Right click (anywhere) | Go back to previous scene (same as Left Arrow). Calls `e.preventDefault()` to suppress context menu. |
| Escape | Exit presentation mode. |
| F5 | Exit presentation mode. |
| D or N | Toggle DM notes overlay (only if current scene has `dmNotes`). |

### Edge Cases

| Scenario | Behavior |
|----------|----------|
| Advance past last scene | No action. Scene counter remains at "N / N". CinematicRenderer stays on final scene. |
| Go back before first scene | No action. Scene counter remains at "1 / N". |
| Scene has no DM notes + D/N pressed | No action. No empty bar shown (D-07). |
| Story has 0 scenes | "Present" button is disabled. Tooltip: "Add scenes to present". |
| Keyboard input while in text input | Not applicable -- presentation mode has no text inputs. All keyboard events are captured. |

### HUD Auto-Hide (D-08)

| Event | Behavior |
|-------|----------|
| Any keypress | HUD becomes visible. Timer resets to 3 seconds. |
| Any mouse movement | HUD becomes visible. Timer resets to 3 seconds. |
| 3 seconds of inactivity | HUD fades out over 400ms using `ease-dissolve` (`cubic-bezier(0.7, 0, 0.84, 0)`). |
| HUD fade-in | 200ms using `ease-unfurl` (`cubic-bezier(0.16, 1, 0.3, 1)`). |

### Cursor Auto-Hide

| Event | Behavior |
|-------|----------|
| Any mouse movement | Cursor becomes visible. Timer resets to 3 seconds. |
| 3 seconds of no mouse movement | Cursor hides via `cursor: none` on the presentation container. |
| Any keypress | Cursor remains hidden (only mouse movement restores it). |

### Exit Flow (D-03)

| Trigger | Behavior |
|---------|----------|
| Escape or F5 | Tauri `invoke("plugin:window|set_fullscreen", { value: false })`. PresentationMode unmounts. StoryEditorPanel's `activeSceneId` is updated to the scene that was showing when the user exited. |

### Scene Replay Behavior (D-12, D-13)

| Direction | Animation Behavior |
|-----------|-------------------|
| Forward (advance) | `playing: true` -- entrance animations play, typewriter narration reveals word-by-word. After all animations complete, scene is in final state. |
| Backward (go back) | `playing: false` -- scene renders in final static state. No entrance animations, narration fully revealed. Instant display. |

## Animation Contract

All animations use the Motion library (v12.38, already installed).

| Animation | Duration | Easing | Implementation |
|-----------|----------|--------|----------------|
| HUD fade-in | 200ms | `ease-unfurl` (0.16, 1, 0.3, 1) | CSS `opacity` transition on HUD container |
| HUD fade-out | 400ms | `ease-dissolve` (0.7, 0, 0.84, 0) | CSS `opacity` transition on HUD container |
| DM notes slide-up (show) | 300ms | `ease-unfurl` (0.16, 1, 0.3, 1) | CSS `transform: translateY(0)` + `opacity: 1` |
| DM notes slide-down (hide) | 250ms | `ease-dissolve` (0.7, 0, 0.84, 0) | CSS `transform: translateY(100%)` + `opacity: 0` |
| Scene transitions | Inherited from CinematicRenderer | Inherited | Crossfade 500ms or fade-to-black 300ms per scene's `TransitionConfig` |
| Entity entrance animations | Inherited from AnimatedEntity | Inherited | Motion `m.div` with path presets |
| Typewriter narration | Inherited from TypewriterNarration | Inherited | Word-by-word reveal per `NarrationSpeed` |
| Cursor hide | Instant | n/a | `cursor: none` class toggle (no animation on cursor itself) |

Note: All CSS transitions use the existing CSS custom properties (`--ease-unfurl`, `--ease-dissolve`) rather than Motion's JavaScript animation for HUD/DM-notes, since these are simple opacity/transform toggles that benefit from GPU compositing without JS overhead.

## Accessibility Contract

| Concern | Implementation |
|---------|----------------|
| Screen reader | Presentation container has `role="region"` and `aria-label="Story presentation"`. Scene changes announced via `aria-live="assertive"` span (inherited from CinematicRenderer). |
| Reduced motion | Check `prefers-reduced-motion: reduce`. If active: skip entrance/exit animations (entities appear in final positions), disable typewriter (narration appears instantly), crossfade transitions reduced to instant cut. |
| Keyboard navigation | All navigation is keyboard-driven by design. Tab key is not used (no focusable elements in presentation). |
| Focus management | On enter: focus moves to presentation container. On exit: focus returns to the "Present" button in StoryEditorPanel. |
| Escape key | Always exits presentation (standard pattern for fullscreen/modal content). |

## Tauri Integration Contract

| API | Usage |
|-----|-------|
| `@tauri-apps/api/core` `invoke()` | Call Rust commands for fullscreen toggle. Pattern: `invoke("plugin:window\|set_fullscreen", { value: true/false })`. Alternatively, use `getCurrentWindow().setFullscreen(true/false)` from `@tauri-apps/api/window` if the plugin approach requires additional setup. |
| `tauri-plugin-window-state` | Already in dependencies. Handles window state persistence. Fullscreen exit should restore previous window dimensions automatically. |

Note: The Tauri window API is used instead of the Web Fullscreen API because WebView2 on Windows intercepts the Escape key before it reaches JavaScript in web fullscreen mode (D-04). The Tauri approach gives JavaScript full control over Escape handling.

## State Management Contract

No new Zustand store is needed. Presentation mode state is local to the `PresentationMode` component:

| State | Type | Scope | Notes |
|-------|------|-------|-------|
| `currentIndex` | `number` | PresentationMode local | Index into sorted scenes array. Initialized from `activeSceneId` on mount. |
| `playing` | `boolean` | PresentationMode local | Whether current scene is animating. Set `true` on forward advance, `false` on backward. |
| `dmNotesVisible` | `boolean` | PresentationMode local | Toggle state for DM notes overlay. Default `false`. |
| `hudVisible` | `boolean` | PresentationMode local | HUD fade state. Driven by inactivity timer. |
| `cursorVisible` | `boolean` | PresentationMode local | Cursor visibility. Driven by mouse movement timer. |
| `isPresenting` | `boolean` | StoryEditorPanel local | Whether presentation mode is active. Controls mount/unmount of PresentationMode via React portal. |

On exit, `PresentationMode` calls `storyStore.setActiveScene(scenes[currentIndex].id)` to sync the editor's active scene (D-03).
