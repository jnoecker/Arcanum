# Phase 12 — UI Design Contract

> Visual and interaction contract for the Showcase Player phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Arcanum design system) |
| Preset | not applicable |
| Component library | none (hand-built components, Tailwind utility classes) |
| Icon library | inline SVG (matching existing showcase pattern -- no icon library dependency) |
| Font | Cinzel (display), Crimson Pro (body), JetBrains Mono (mono) via @fontsource |

**Note:** The showcase has its own lighter-palette variation of the Arcanum design system defined in `showcase/src/index.css`. All new components MUST use the showcase's `@theme` tokens (e.g., `bg-bg-primary`, `text-accent`), NOT the creator's darker tokens.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, thumbnail strip gaps |
| sm | 8px | Compact element spacing, control bar button gaps |
| md | 16px | Default element spacing, card grid gaps, control bar padding |
| lg | 24px | Section padding, story card content padding |
| xl | 32px | Player area vertical margins, page section gaps |
| 2xl | 48px | Stories page top/bottom padding |
| 3xl | 64px | Not used in this phase |

Exceptions: Control bar buttons use 40px touch targets (10px padding around 20px icons) for comfortable click/tap interaction. This is not a multiple of 4 but aligns with WCAG touch target minimums.

| Role | Token / Value | Usage |

**Accent reserved for:**
- "Stories" nav link when active
- Play/pause toggle icon in active state

- Scene counter current number
- Auto-play progress indicator

**Player viewport:** The 16:9 CinematicRenderer container uses `bg-black` (#000000) as its internal background (inherited from the ported renderer), distinct from the showcase's deep-space palette.

**Control bar background:** `bg-bg-secondary/80 backdrop-blur-md` -- semi-transparent with blur for depth without obscuring page context.

## Interaction Contract

### Navigation Modes

| Mode | Behavior | Keyboard | Mouse/Touch |
|------|----------|----------|-------------|
| Click-through (default) | Manual scene advancement | Right arrow / Space / Enter = next; Left arrow = prev; Escape = exit to listing | Click renderer area = next |
| Auto-play | Timed advancement with manual override | Same as click-through; manual nav resets timer | Same as click-through |
| Scroll | Vertical scroll with snap-to-scene | Not applicable (native scroll) | Scroll / swipe (native CSS scroll-snap) |

### Auto-play Timer

- Presets: 5s, 10s (default), 15s
- Timer starts after scene entrance animations complete
- Manual navigation (keyboard or click) resets the timer for the current scene
- Switching to a different mode pauses the timer
- At the last scene: timer stops (no auto-loop)
- Visual indicator: subtle progress bar at the bottom of the CinematicRenderer, `h-0.5 bg-accent/60`, width animates from 0% to 100% over the interval duration using CSS `transition: width {interval}ms linear`

### Scene Replay Behavior

- Navigating forward: entrance animations play (entity paths, typewriter narration)
- Navigating backward: scene shows its final state (no re-animation). Consistent with Phase 11 decision D-12
- In scroll mode: scrolling to a new scene triggers entrance animations via IntersectionObserver. Scrolling back shows final state

### Mode Switcher

- Pattern: segmented toggle group (three options in a row)
- Container: `flex rounded-lg border border-border-muted/40 overflow-hidden`
- Each option: `px-3 py-1.5 text-[11px] font-display tracking-[0.12em] uppercase`
- Active state: `bg-accent/12 text-accent`
- Inactive state: `text-text-muted hover:text-text-secondary`
- Transition: `transition-colors duration-200`
- Labels: "Click" | "Auto" | "Scroll"

### Timing Dropdown

- Only visible when Auto-play mode is active
- Pattern: native `<select>` styled with Arcanum tokens (matching the showcase's input styling)
- Options: "5s", "10s", "15s"
- Background: `bg-bg-secondary/60 border border-border-muted/60 rounded-lg`
- Text: 11px Cinzel, `text-text-primary`

### "Play Story" Button on Article Pages

- Appears on ArticlePage when `article.template === "story"` (requires extending the ArticleTemplate type)
- Position: below the article header, above the content. Full-width banner style
- Layout: `flex items-center justify-center gap-3 py-4 rounded-lg border border-accent/30 bg-accent/6 hover:bg-accent/12 transition-colors duration-300`
- Icon: inline SVG play triangle (16px), `text-accent`
- Text: "Play Story" in Cinzel 13px, `text-accent`, tracking 0.16em, uppercase
- Links to `/stories/{storyId}` using React Router `<Link>`

## State Specifications

### StoryPlayer State Machine

```
States: idle | playing | paused | complete

Transitions:
  idle -> playing       : user clicks play or advances scene
  playing -> paused     : user clicks pause (auto-play mode only)
  playing -> playing    : scene auto-advances (auto-play timer fires)
  playing -> complete   : last scene animations finish
  paused -> playing     : user clicks play
  complete -> playing   : user navigates to previous scene
  * -> idle             : user switches mode
```

### Data Flow

```
ShowcaseData.stories[] -> useShowcase() -> StoryPlayerPage
  -> StoryPlayer (manages currentIndex, playbackState, mode)
    -> CinematicRenderer (receives scenes, currentIndex, playing)
    -> PlayerControlBar (receives/dispatches mode, index, play state)
```

### URL State

- `/stories` -- stories listing
- `/stories/:id` -- story player, starts at scene 0 in click-through mode
- No scene index in URL (scene state is ephemeral)
- No mode persistence in URL (always resets to click-through on page load)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Arrow keys, Space, Enter, Escape work in click-through and auto-play modes |
| Screen reader | CinematicRenderer has `role="region" aria-label="Scene playback"` with `aria-live="assertive"` scene announcements (inherited from ported component) |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all animations (inherited from showcase index.css) |
| Focus management | Control bar buttons have `focus-visible:ring-2 focus-visible:ring-accent/40` |
| Scene counter | Live region: `aria-live="polite"` on scene counter text for screen reader updates |
| Mode switcher | `role="radiogroup"` with `role="radio"` and `aria-checked` on each option |
| Touch targets | All control bar buttons minimum 40px touch target |
| Skip link | Existing showcase skip-to-content link covers the player page |
| Alt text | Cover images on story cards: `alt="{story title}"`. Scene room images: `alt=""` (decorative, narration provides context) |

## Animation Contract

All animations use the Motion library (to be added to showcase dependencies).

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Scene crossfade | 500ms | cubic-bezier(0.4, 0, 0.2, 1) | Scene index changes |
| Scene fade-to-black | 300ms (out) + 300ms (in) | cubic-bezier(0.4, 0, 0.2, 1) | Scene index changes |
| Entity entrance | 600ms | cubic-bezier(0.16, 1, 0.3, 1) | Scene becomes active + playing |
| Typewriter narration | ~50ms per word (normal speed) | linear | Scene becomes active + playing, 200ms delay |
| Auto-play progress bar | {interval}ms | linear | Auto-play timer starts |
| Story card hover zoom | 700ms | ease-out | Mouse enter on story card image |
| Page entrance | 200ms | ease-out | Route change (inherited from Layout.tsx) |
| Card stagger | 400ms per card, 40ms stagger | cubic-bezier(0.16, 1, 0.3, 1) | StoriesPage mount |
| Control bar fade | 200ms | ease-out | Mode switch hides/shows control bar |

## Responsive Behavior

| Breakpoint | StoriesPage Grid | Player Width | Control Bar |
|------------|-----------------|--------------|-------------|
| < 640px (mobile) | 1 column | Full width minus 20px padding | Stacks: nav row + mode row |
| 640-1023px (tablet) | 2 columns | Full width minus 32px padding | Single row, compact |
| >= 1024px (desktop) | 3 columns | `max-w-4xl` (896px) centered | Single row, full labels |

### Mobile Control Bar Stack

On screens < 640px, the control bar splits into two rows:
- Top row: prev, scene counter, next, play/pause
- Bottom row: mode switcher, timing dropdown

Container becomes `flex flex-col gap-2 sm:flex-row sm:items-center sm:gap-3`.

### Scroll Mode on Mobile

Native CSS scroll-snap with touch/swipe works without custom gesture code (D-13). Scene containers maintain 16:9 aspect ratio. Control bar is hidden. A subtle scroll indicator (thin line at bottom of viewport, `h-0.5 bg-accent/20`) hints at scrollability.
